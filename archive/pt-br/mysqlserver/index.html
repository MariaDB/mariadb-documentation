mysql.server

O script de inicialização mysql.server se encontra incluido nas distribuições MariaDB e MySQL no Unix. Pode ser usado em sistemas como o Linux e o Solaris que usa diretórios de execução tipo System V para iniciar e parar serviços de sistema. Também é usado pelo item de inicialização do MariaDB no Mac OS X.

mysql.server pode ser encontrado no diretório de arquivos de suporte no diretório de instalação do MariaDB ou numa distribuição fonte deste.

Se você estiver usando o pacote RPM do servidor Linux (MariaDB-server-VERSION.rpm), o script mysql.server será instalado no diretório /etc/init.d com o nome de mysql . Não precisa de instalá-lo manualmente.

Alguns fornecedores oferecem pacotes RPM que instalam um script de inicialização com um nome diferente como mysqld.

Se você instalar o MariaDB desde uma distribuição fonte, ou usando um formato de distribuição binária que não instala o mysql.server automaticamente, você então pode instalá-lo automaticamente. Instruções são dadas em Iniciando e Parando MariaDB automaticamente.

mysql.server lê opções vindas das seções [mysql.server] e [mysqld] dos arquivos de opções. Para compatibilidade com versões anteriores, também lê seções [mysql_server] embora você deve renomear essas seções para [mysql.server] quando estiver usando MySQL 5.0.

mysql.server suporta as seguintes opções:

  • --basedir=path
    • O caminho para o diretório de instalação do MariaDB.
  • --datadir=path
    • O caminho para o diretório de dados do MariaDB..
  • --pid-file=file_name
    • O nome do caminho do arquivo no qual o servidor deveria escrever seu propio ID de processo.
  • --service-startup-timeout=file_name
    • Quanto tempo em segundos tem que esperar para confirmação de inicialização do servidor. Se o servidor não iniciar durante esse tempo, mysql.server termina com um erro. O valor padrão é 900. Um valor 0 significa não ter que esperar para a inicialização. Valores negativos significam esperar para sempre (sem tempo limite). Esta opção foi adicionada no MySQL 5.0.40. Antes disso, o valor 900 era sempre usado.
  • --use-mysqld_safe
    • Use mysqld_safe para iniciar o servidor. Este é o padrão. Esta opção foi adicionada no MySQL 5.0.4.
  • --use-manager
    • Use o Gerenciador de Instâncias para iniciar o servidor. Esta opção foi adicionada no MySQL 5.0.4.
  • --user=user_name
    • O nome de usuário a usar para executar mysqld. Esta opção foi adicionada no MySQL 5.0.4.
