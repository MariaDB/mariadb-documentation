aria_read_log

aria_read_log è uno strumento utile per mostrare o applicare le voci del log delle transazioni di Aria.

Nota: Aria è compilato senza -DIDENTICAL_PAGES_AFTER_RECOVERY, il che significa che i file delle tabelle non sono identici byte-per-byte ai file creati durante una normale esecuzione. Questo non dovrebbe essere un problema, eccetto il caso degli script di test che tentano di confrontare i file prima e dopo il recupero.

Utilizzo: aria_read_log OPZIONI

E' necessario specificare -d oppure -a

Opzioni

OpzioneSpiegazione
-a, --applyApplica il log alle tabelle: modifica le tabelle! Prima bisognerebbe fare un backup! Mostra parecchie informazioni, se non è specificato --silent.
--character-sets-dir=nomeLa directory dove si trovano i set di caratteri.
-c, --checkSe viene usato --display-only, verifica che i record siano completamente leggibili (usato per il debug).
-?, --helpMostra l'help e termina.
-d, --display-onlyMostra brevi informazioni lette dalle intestazioni dei record.
-e, --end-lsn=#Smette di applicare i record a partire da questo lsn. Se si usa end-lsn, UNDO:s non sarà applicato
-h, --aria-log-dir-path=nomePercorso della directory dove devono essere memorizzati i log delle transazioni
-P, --page-buffer-size=#Le dimensioni del buffer usato per i blocchi degli indici delle tabelle Aria.
-o, --start-from-lsn=#Inizia a leggere il log da questo lsn.
-C, --start-from-checkpointInizia ad applicare i record dall'ultimo checkpoint.
-s, --silentScrive meno informazioni durante la fare di applicazione o annullamento.
-T, --tables-to-redo=nomeElenco di tabelle separate da virgola alle quali deve essere applicato REDO. Da usare solo se si desidera recuperare alcune tabelle.
-t, --tmpdir=nomePercorso dei file temporanei. E' possibile specificare più percorsi, separati dai due punti (:)
--translog-buffer-size=#Le dimensioni del buffer usato per il log delle transazioni per le tabelle Aria.
-u, --undoApplica i record UNDO alle tabelle. Si disabilita con --disable-undo. E' attivo per default; per disabilitarlo: --skip-undo to disable.
-v, --verboseScrive più informazioni durante la fase di applicazioni e annullamento.
-V, --versionMostra la versione.

Variabili

E' possibile impostare le seguenti variabili passandole come opzioni ad aria_read_log attraverso la riga di comando, o impostarle nella sezione [aria_read_log] del file my.cnf.

OpzioneValore predefinito
applyFALSE
character-sets-dirNessun default
checkFALSE
display-onlyFALSE
end-lsn0
aria-log-dir-path.
page-buffer-size268427264
start-from-lsn0
start-from-checkpointFALSE
silentFALSE
tmpdirNiente
translog-buffer-size2097152.
undoON
verboseFALSE
