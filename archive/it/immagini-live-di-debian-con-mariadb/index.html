Immagini Live di Debian con MariaDB

Un membro della comunità di MariaDB, Mark <ms (at) it-infrastrukturen (dot) org>, ha creato le immagini ISO live basate su Debian "squeeze" 6.0.4, con MariaDB 5.2 o 5.3 preinstallato.

Queste immagini live e installabili possono essere utilizzate per eseguire velocemente un server MariaDB server in modalità live per imparare, per testarlo, o per semplificare e velocizzare le installazioni off-line su hard disk dei server MariaDB basati su Debian.

Il sistema si avvia in modalità live da una memoria usb (o CD/DVD) e si esegue in memoria RAM senza accedere all'hard disk.

La stessa memoria usb (o CD/DVD) può essere utilizzata per installare un server su disco, utilizzando l'installer di Debian.

Tutti i pacchetti MariaDB necessari sono inclusi, quindi non è necessaria una connessione ad internet.

Sono disponibili tre tipi di immagini: testuale (riga di comando), LXDE e Gnome. Le immagini testuali possono essere utilizzate per i test o per le installazioni off-line. I due tipi di GUI, LXDE e Gnome, possono essere usati per imparare, per testare in modalità live, o per le installazioni desktop for off-line. Le immagini live di Debian con LXDE (Gnome, KDE o altro) sono abbastanza comode per un uso quotidiano comune, sostituibile a qualsiasi altro OS.

Vi sono tre immagini per ogni tipo: una per i sistemi a 32-bit (i386), una per i sistemi a 64-bit (amd64) e una con entrambi.

  1. Immagini iso MariaDB 5.2 Live (testuali) per i386, amd64 o multi-architettura.
    • binary-hybrid-squeeze-i386-mariadb52-text.iso
    • binary-hybrid-squeeze-amd64-mariadb52-text.iso
    • binary-hybrid-squeeze-i386-amd64-mariadb52-text.iso
  1. Immagini iso MariaDB 5.3 Live (testuali) per i386, amd64 o multi-architettura.
    • binary-hybrid-squeeze-i386-mariadb53-text.iso
    • binary-hybrid-squeeze-amd64-mariadb53-text.iso
    • binary-hybrid-squeeze-i386-amd64-mariadb53-text.iso
  1. Immagini iso MariaDB 5.3 Live con LXDE per i386, amd64 o multi-architettura.
    • binary-hybrid-squeeze-i386-mariadb53-lxde.iso
    • binary-hybrid-squeeze-amd64-mariadb53-lxde.iso
    • binary-hybrid-squeeze-i386-amd64-mariadb53-lxde.iso
  1. Immagini iso MariaDB 5.3 Live con Gnome per i386, amd64 o multi-architettura.
    • binary-hybrid-squeeze-i386-mariadb53-gnome.iso
    • binary-hybrid-squeeze-amd64-mariadb53-gnome.iso
    • binary-hybrid-squeeze-i386-amd64-mariadb53-gnome.iso

Le immagini LXDE e Gnome contengono la documentazione in /srv/PDF. Vi sono anche le istruzioni su come creare immagini live di Debian in modalità live (occorrono almeno 16GB RAM per poterlo fare). Si vedano i file README, README.live e live-manual.en.pdf in /srv/PDF per ulteriori dettagli.

Nota: Alcuni notebook HP non sono in grado di avviarsi usando immagini iso ibride binarie da una memoria USB.

Download

Per ottenere le immagini iso si può utilizzare rsync:

rsync -avP rsync://rsync.it-infrastrukturen.org/ftp/public-mariadb/<image_or_file_name_as_above> .

oppure visitare: http://rsync.it-infrastrukturen.org/mariadb/

Preparare un media avviabile

Una memoria USB o un CD/DVD possono essere usati come media avviabili. Il modo migliore è usare una memoria USB.

  • Su Linux: dd if=./<image_name_as_above.iso> of=/dev/<USB-stick-ID_like_sdb_or_sdc>
  • Altri sistemi: cygwin include dd e altri strumenti linux/Unix.

Le immagini iso images sono state testate con successo su:

  • I netbook Acer ASPIRE @ne (1GB RAM)
  • I notebook ThinkPad T60p e T61p (2 e 4GB RAM)
  • Desktop Xeon E31270 / Asus P8BWS (16GB ECC RAM)
  • Desktop AMD FX-4100 / Asus M5A99X EVO (16GB ECC RAM)
  • I server HP DL385g7 Opteron 6170SE e 6180SE (32 or 64GB ECC RAM)

Note

E' possibile specificare alcune opzioni alla riga di avvio per scopi particolari, per esempio per installare pacchetti aggiuntivi da un repository locale sull'immagine, oppure per assegnare un indirizzo IP statico ed eseguire sshd.

  1. Per configurare un IP statico: "ip=". Esempio: "ip=eth0:192.168.1.53:255.255.255.0:192.168.1.1"
    • Il formato completo di "ip=" è: "ip=[interfaccia]:[indirizzo_IP]:[machera_di_rete]:[gateway]"
  1. Se si usa un valore "ip=" vuoto, verrà impostato al contenuto di /etc/network/interfaces e /etc/resolv.conf. Se l'opzione "ip=" viene omessa, verrà usato dhcp per ottenere un ip.
  1. Per installare ed eseguire sshd, così che l'immagine possa agire come server in un ambiente di test: "sshd=on"
    • L'utente live è "sql" e la password è "live". E' bene cambiarla subito dopo il primo login!
  1. A seconda dell'immagine utilizzata, viene installato un server MariaDB 5.2 o 5.3. Non vi è alcuna password di root preimpostata, perciò è bene crearne una subito dopo il primo login!
  1. Eseguendo repo-off-line.sh si attivano i repository locali dell'immagine per apt-get (installazioni off-line) o repo-on-line.sh (per usare i repository in rete).
  1. Le immagini iso live binarie ibride con scelte multiple nel menu di boot consentono di assegnare due indirizzi IP differenti - per esempio, per testare le operazioni simultanee sui database.
