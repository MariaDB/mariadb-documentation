SHOW VARIABLES

Sintassi

SHOW [GLOBAL | SESSION] VARIABLES
    [LIKE 'pattern' | WHERE espr]

Spiegazione

SHOW VARIABLES mostra i valori delle variabili di sistema di MariaDB. Queste informazioni possono essere ottenute anche con il comando mysqladmin variables. La clausola LIKE, se presente, restituisce solo le variabili i cui nomi corrispondono al pattern. La clausola WHERE può essere usata per specificare condizioni più generiche.

Con il modificatore GLOBAL, SHOW VARIABLES restituisce i valori che si applicano alle nuove connessioni verso MariaDB. Con SESSION, mostra i valori relativi alla connessione corrente. Se nessun modificatore viene specificato, quello predefinito è SESSION. LOCAL è sinonimo di SESSION.

Con la clausola LIKE, l'istruzione restituisce solo le variabili i cui nomi corrispondono al pattern. Per ottenere una riga per una variabile specifica, si usi LIKE in questo modo:

SHOW VARIABLES LIKE 'maria_group_commit';
SHOW SESSION VARIABLES LIKE 'maria_group_commit';

Per ottenere un elenco di variabili i cui nomi corrispondono a un pattern, si usi il carattere jolly "%" nella clausola LIKE:

SHOW VARIABLES LIKE '%maria%';
SHOW GLOBAL VARIABLES LIKE '%maria%';

I caratteri jolly possono essere utilizzati in qualsiasi posizione all'interno del pattern. Strettamente parlando, siccome "_" è un jolly che corrisponde a un qualsiasi carattere singolo, bisognerebbe precederlo con un escape ("\_") per trovare una corrispondenza letterale. In pratica, raramente è necessario.

Si veda SET per ulteriori informazioni su come impostare le variabili di sistema del server.

Alla pagina Variabili di sistema del server sono elencate tutte le variabili che possono essere impostate.
