Panoramica su SHOW

Spiegazione

SHOW ha molte forme che forniscono diverse informazioni sui database, tabelle, colonne, o sullo stato del server. Questa sezione descrive le seguenti:

  • SHOW AUTHORS
  • SHOW CHARACTER SET [like_o_where]
  • SHOW COLLATION [like_o_where]
  • SHOW [FULL] COLUMNS FROM nome_tabella [FROM nome_db] [like_o_where]
  • SHOW CONTRIBUTORS
  • SHOW CREATE DATABASE nome_db
  • SHOW CREATE EVENT nome_evento
  • SHOW CREATE PROCEDURE nome_proc
  • SHOW CREATE TABLE nome_tabella
  • SHOW CREATE TRIGGER nome_trigger
  • SHOW CREATE VIEW nome_vista
  • SHOW DATABASES [like_o_where]
  • SHOW ENGINE nome_engine {STATUS | MUTEX}
  • SHOW [STORAGE] ENGINES
  • SHOW ERRORS [LIMIT [offset,] num_righe]
  • SHOW [FULL] EVENTS
  • SHOW FUNCTION CODE nome_funz
  • SHOW FUNCTION STATUS [like_o_where]
  • SHOW GRANTS FOR utente
  • SHOW INDEX FROM nome_tabella [FROM nome_db]
  • SHOW INNODB STATUS
  • SHOW OPEN TABLES [FROM nome_db] [like_o_where]
  • SHOW PLUGINS
  • SHOW PROCEDURE CODE nome_proc
  • SHOW PROCEDURE STATUS [like_o_where]
  • SHOW PRIVILEGES
  • SHOW [FULL] PROCESSLIST
  • SHOW PROFILE [tipi] [FOR QUERY n] [OFFSET n] [LIMIT n]
  • SHOW PROFILES
  • SHOW [GLOBAL | SESSION] STATUS [like_o_where]
  • SHOW TABLE STATUS [FROM nome_db] [like_or_where]
  • SHOW TABLES [FROM nome_db] [like_o_where]
  • SHOW TRIGGERS [FROM nome_db] [like_o_where]
  • SHOW [GLOBAL | SESSION] VARIABLES [like_o_where]
  • SHOW WARNINGS [LIMIT [offset,] num_righe] 
like_o_where:
    LIKE 'pattern'
  | WHERE espr

Se la sintassi di una data istruzione SHOW comprende una clausola LIKE 'pattern', 'pattern' è una stringa che contiene i caratteri jolly SQL "%" e "_". Il pattern è utile per restringere l'output a certi elementi.

Diversi comandi SHOW accettano anche una clausola WHERE che permette una maggiore flessibilità nello specificare quali righe debbano essere mostrate. Si veda http://dev.mysql.com/doc/refman/5.1/en/extended-show.html per ulteriori informazioni.
