Start with the slow query log. Use `EXPLAIN` and start analyzing them.</p> <p>With a large amount of writes ensure `innodb_log_file_size`/ `innodb_log_files_in_group` are increased. `SHOW GLOBAL STATUS` along with your configuration will be useful. MariaDB's error log may also be useful.</p> <p>Regarding hanging check table cache variables and ensure they are sufficient for you active workload.</p> <p>Perhaps seeking help from a consultant would be a prudent investment.</p> Daniel Blackhttps://mariadb.com/kb/ko/how-to-tune-the-huge-memory-system/+comments/4324
