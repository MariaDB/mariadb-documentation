MariaDB Knowledge Base Comments for: How to store the string in %shttps://mariadb.com/kb/en/how-to-store-the-string-in-s/+comments/feed/en-usTue, 28 Nov 2023 02:46:39 +0000Re: How to store the string in %shttps://mariadb.com/kb/en/how-to-store-the-string-in-s/+comments/1873<p>I do not know of your specific case, but this looks like a format argument to the C printf(3) function.</p> <p>If you are on UNIX or Linux, I suggest you type "man 3 printf" at a command prompt. that will show (in part): "The format string is composed of zero or more directives: ordinary characters (not %), which are copied unchanged to the output stream; and conversion specifications, each of which results in fetching zero or more subsequent arguments. Each conversion specification is introduced by the % character."</p> <p>If it is indeed a printf(3) format string, "%s" would mean that a corresponding argument to printf(3) is to be interpreted as a character array, and the "%d" would mean that the corresponding argument is interpreted as a decimal number.</p> <p>So the function call might look something like this: "printf('%s, %d', databasename, #ENOENT)". There is a list of common error codes in /usr/include/sys/errno.h .</p> Jan Steinmanhttps://mariadb.com/kb/en/how-to-store-the-string-in-s/+comments/1873
