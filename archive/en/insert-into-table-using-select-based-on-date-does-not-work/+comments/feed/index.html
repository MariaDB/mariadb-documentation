Even the original query gives a warning in this case:</p> <pre class="fixed">select STR_TO_DATE('20200809 020917076','%Y%m%d %H%i%s'); +---------------------------------------------------+ | STR_TO_DATE('20200809 020917076','%Y%m%d %H%i%s') | +---------------------------------------------------+ | 2020-08-09 02:09:17 | +---------------------------------------------------+ 1 row in set, 1 warning (0.000 sec) Warning (Code 1292): Truncated incorrect datetime value: '20200809 020917076' </pre><p>You can use <code>%f</code> to format for microseconds.</p> <pre class="fixed">select STR_TO_DATE('20200809 020917076','%Y%m%d %H%i%s%f'); +-----------------------------------------------------+ | STR_TO_DATE('20200809 020917076','%Y%m%d %H%i%s%f') | +-----------------------------------------------------+ | 2020-08-09 02:09:17.076000 | +-----------------------------------------------------+ </pre>Ian Gilfillanhttps://mariadb.com/kb/en/insert-into-table-using-select-based-on-date-does-not-work/+comments/5483
