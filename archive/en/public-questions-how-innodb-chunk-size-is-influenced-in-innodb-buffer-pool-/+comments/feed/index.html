That's it. Previous defaults where too small meaning to took a long time to initialize on large memory systems, and who was really going to downsize by 0.0001%. The buffer_pool_size/64 puts in the the reasonable number percentage for a quick resize if needed. So provided the buffer_pool_size intially is in the order of magnitude of the current buffer pool size, the default is sufficient</p> <p>Also chunks are being removed - <a href="https://jira.mariadb.org/browse/MDEV-29445">MDEV-29445</a>.</p> <p>InnoDB pages are always the fixed size, default 16k but can be down to 4k or up to 64k (configurable installation time only).</p> Daniel Blackhttps://mariadb.com/kb/en/public-questions-how-innodb-chunk-size-is-influenced-in-innodb-buffer-pool-/+comments/6625
