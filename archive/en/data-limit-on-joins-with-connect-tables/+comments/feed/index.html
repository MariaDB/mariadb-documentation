It is to create a table that make the remote server to the join. Here is an example:</p> <p>create table categories engine=connect table_type=ODBC block_size=10 Connection='DSN=MS Access Database;DBQ=C:/Program Files/Microsoft Office/Office/1033/FPNWIND.MDB;';</p> <p>create table products engine=connect table_type=ODBC block_size=10 Connection='DSN=MS Access Database;DBQ=C:/Program Files/Microsoft Office/Office/1033/FPNWIND.MDB;';</p> <p>If you do:</p> <p>select productid, productName, p.categoryid, categoryname, description from products p natural join categories;</p> <p>MariaDB has to retrieve the relevant parts of the two tables and make the join locally. This is not good performance wise (except for small tables) and can be limited by the used storage. To have the join done by the data source you can do:</p> <p>create table prodcat engine=connect table_type=ODBC block_size=10 Connection='DSN=MS Access Database;DBQ=C:/Program Files/Microsoft Office/Office/1033/FPNWIND.MDB;' srcdef='select p.ProductID, p.ProductName, p.CategoryID, c.CategoryName, c.Description from products p inner join categories c on p.CategoryID=c.CategoryID';</p> <p>This table is based on a query specified by the SRCDEF option that will be executed remotely by the data source (note that the syntax must be the data source syntax) Now you can get the result of the join by:</p> <p>select * from prodcat;</p> <p>In your case, perhaps this will avoid loosing result rows.</p> Olivier Bertrandhttps://mariadb.com/kb/en/data-limit-on-joins-with-connect-tables/+comments/1116Re: Data limit on joins with connect tables?https://mariadb.com/kb/en/data-limit-on-joins-with-connect-tables/+comments/1038<p>Probably this won't help you, but... try checking the value of <a href="/kb/en/server-system-variables/#sql_select_limit">sql_select_limit</a>.
