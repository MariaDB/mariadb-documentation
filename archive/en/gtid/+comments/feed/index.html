So gtid_strict_mode=ON should be employed to avoid this.</p> <div style="margin-left:2em"><p> How is the stream order affected by out-of-order parallel replication where the slave has its own slaves with log-slave-updates=1?</p> </div> <p>Stream order for transactions in the same replication domain is not affected, they will always be in the same order in the slave binlog as in the master binlog with respect to each other.</p> <p>Transactions in different replication domains can appear in a different order with respect to each other in the slave binlog, compared to the order on the master.</p> <p>Effectively, each replication domain is logged as an independent stream.
