MariaDB Knowledge Base Comments for: Aria LIMIT clause performancehttps://mariadb.com/kb/en/aria-limit-clause-performance/+comments/feed/en-usMon, 23 Dec 2024 12:06:48 +0000Re: Aria LIMIT clause performancehttps://mariadb.com/kb/en/aria-limit-clause-performance/+comments/72<p>If the issue is that you are using a high start point for the rows, there is no way to improve the speed for this. The reason is that as rows are not ordered, there is no way to start from a certain row other than reading through all the old rows.</p> <p>A better way to do this is to select rows ordered by a key field and the next query from where you left off.</p> <p>Example:</p> <pre class="fixed">SELECT * from t1 where key &gt;= '' ORDER BY key LIMIT 1000 SELECT * from t1 where key &gt; 'last-fetched-value' ORDER BY key LIMIT 1000 </pre>Michael Wideniushttps://mariadb.com/kb/en/aria-limit-clause-performance/+comments/72
