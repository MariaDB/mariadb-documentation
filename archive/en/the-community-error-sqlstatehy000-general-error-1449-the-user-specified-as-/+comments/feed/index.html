I personnaly had the same message when I tried to create a view that relied on other views that did not exist in the schema or which definer user was not recognised by Mariadb. The solution is quite obvious: 1) Ensure that the definer user exists in your database. Create him if needed and do not forget to flush privileges. 2) Create or alter the dependances views with the correct definer user. 3) Create or alter the view for which the message occured with the correct definer user.</p> <p>Sorry for Gene.
