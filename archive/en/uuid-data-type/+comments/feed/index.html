MariaDB Knowledge Base Comments for: UUID Data Typehttps://mariadb.com/kb/en/uuid-data-type/+comments/feed/en-usMon, 29 Jul 2024 04:20:35 +0000Re: Function to simulatehttps://mariadb.com/kb/en/uuid-data-type/+comments/6785<p>The functions above provided by Johannes gave me cause for concern about the consistency of the new <code class="fixed" style="white-space:pre-wrap">uuid</code> data type with existing usage of the <code class="fixed" style="white-space:pre-wrap">BINARY(16)</code> data type. Eventually I worked out that, unfortunately, the functions he provided are <strong>not symmetric, and inconsistent with how the new uuid data type is represented</strong>.</p> <p>Taking an example query from a database table I have, where:</p> <ul start="1"><li>the <code class="fixed" style="white-space:pre-wrap">uuid_as_binary16</code> column has data type <code class="fixed" style="white-space:pre-wrap">BINARY(16)</code> </li><li>all <code class="fixed" style="white-space:pre-wrap">SELECT</code> queries shown below retrieve data from the same pre-existing row </li><li>the <code class="fixed" style="white-space:pre-wrap">uuid</code> column has data type <code class="fixed" style="white-space:pre-wrap">UUID</code> and was populated by running <code class="fixed" style="white-space:pre-wrap">UPDATE oliver_test SET uuid = uuid_as_binary16;</code> </li></ul> <pre class="fixed">+--------------------------------------------------+ | HEX(UUID_TO_BIN(BIN_TO_UUID2(uuid_as_binary16))) | +--------------------------------------------------+ | 3FEBCBE0500DAF0E4CDFFB19BDBB4847 | +--------------------------------------------------+ +----------------------------------+ | HEX(uuid_as_binary16) | +----------------------------------+ | 3FEBCBE0500D4CDFAF0EFB19BDBB4847 | +----------------------------------+ </pre><p>After some research I came across <a href="https://gist.github.com/jamesgmarks/56502e46e29a9576b0f5afea3a0f595c">https://gist.github.com/jamesgmarks/56502e46e29a9576b0f5afea3a0f595c</a> which provided alternative functions (reprinted here in case the link target gets changed):</p> <pre class="fixed">DELIMITER // CREATE FUNCTION BIN_TO_UUID(b BINARY(16)) RETURNS CHAR(36) BEGIN DECLARE hexStr CHAR(32); SET hexStr = HEX(b); RETURN LOWER(CONCAT( SUBSTR(hexStr, 1, 8), '-', SUBSTR(hexStr, 9, 4), '-', SUBSTR(hexStr, 13, 4), '-', SUBSTR(hexStr, 17, 4), '-', SUBSTR(hexStr, 21) )); END// CREATE FUNCTION UUID_TO_BIN(uuid CHAR(36)) RETURNS BINARY(16) BEGIN RETURN UNHEX(REPLACE(uuid, '-', '')); END// DELIMITER ; </pre><p>Using these functions provides symmetrical results:</p> <pre class="fixed">+-------------------------------------------------+ | HEX(UUID_TO_BIN(BIN_TO_UUID(uuid_as_binary16))) | +-------------------------------------------------+ | 3FEBCBE0500D4CDFAF0EFB19BDBB4847 | +-------------------------------------------------+ +----------------------------------+ | HEX(uuid_as_binary16) | +----------------------------------+ | 3FEBCBE0500D4CDFAF0EFB19BDBB4847 | +----------------------------------+ </pre><p>Also, crucially, these functions (unlike the ones provided by Johannes) provide a representation that is consistent with how the new data type works. Initially I had run:</p> <pre class="fixed">UPDATE oliver_test SET uuid = uuid_as_binary16; </pre><p>Then I could compare the two datatypes:</p> <pre class="fixed">+--------------------------------------+ | BIN_TO_UUID(uuid_as_binary16) | +--------------------------------------+ | 3febcbe0-500d-4cdf-af0e-fb19bdbb4847 | +--------------------------------------+ +--------------------------------------+ | uuid | +--------------------------------------+ | 3febcbe0-500d-4cdf-af0e-fb19bdbb4847 | +--------------------------------------+ </pre><p>I hope this is useful to others.</p> Oliver Lockwoodhttps://mariadb.com/kb/en/uuid-data-type/+comments/6785Function to simulatehttps://mariadb.com/kb/en/uuid-data-type/+comments/6640<p>To simulate the new UUID type insert and retrieve to a BINARY(16) format these functions might come in handy:</p> <p>to convert (and store) a UUID in a 16 byte binary</p> <pre class="fixed">CREATE FUNCTION `UUID_TO_BIN`(uuid char(36)) RETURNS binary(16) BEGIN RETURN UNHEX( CONCAT( SUBSTRING(uuid, 25, 12), SUBSTRING(uuid, 20, 4), SUBSTRING(uuid, 15, 4), SUBSTRING(uuid, 10, 4), SUBSTRING(uuid, 1, 8) )); END </pre><p>To retrieve and convert back to UUID</p> <pre class="fixed">CREATE FUNCTION `BIN_TO_UUID2`(b binary(16)) RETURNS char(36) CHARSET ascii BEGIN DECLARE hexStr CHAR(32); SET hexStr = HEX(b); RETURN LOWER(CONCAT( SUBSTR(hexStr, 25, 12), '-', SUBSTR(hexStr, 21, 4), '-', SUBSTR(hexStr, 13, 4), '-', SUBSTR(hexStr, 17, 4), '-', SUBSTR(hexStr, 1, 12) )); END </pre>Johannes Kingmahttps://mariadb.com/kb/en/uuid-data-type/+comments/6640
