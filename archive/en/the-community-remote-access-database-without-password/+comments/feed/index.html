MariaDB Knowledge Base Comments for: Remote access database without passwordhttps://mariadb.com/kb/en/the-community-remote-access-database-without-password/+comments/feed/en-usMon, 29 Jan 2024 21:09:38 +0000Re: Remote access database without passwordhttps://mariadb.com/kb/en/the-community-remote-access-database-without-password/+comments/6491<p>As the unix_socket is on the filesystem it is inherently not remote.</p> <p>You can tunnel via ssh to make a local connection, tcp or unix, appear to the database as if its coming from the unix socket.</p> <p>This is done by:</p> <pre class="fixed">ssh -L /tmp/mylocalsocket.socket:/run/mysqld/mysqld.sock user@mydatabaseserver.address </pre><p>Then MariaDB connections to /tmp/mylocalsocket will end up being transported over ssh and appear as a connection to the database socket, just as if it was local.</p> <p>As a tcp option:</p> <pre class="fixed">ssh -L localhost:3306:/run/mysqld/mysqld.sock user@mydatabaseserver.address </pre><p>This will make port 3306 connections locally appear at the unix socket.</p> <p>Note for both cases it must be the "user" that is granted the unix_socket authentication on the database.</p> Daniel Blackhttps://mariadb.com/kb/en/the-community-remote-access-database-without-password/+comments/6491
