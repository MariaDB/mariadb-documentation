Therefore, below changes needs to be done in SE linux policy</p> <pre class="fixed">1) semanage fcontext -a -t mysqld_db_t &#34;/mnt/volume-nyc1-01/mysql(/.*)?&#34; 2) restorecon -Rv /mnt/volume-nyc1-01/mysql 3) Also add below section in mariadb configuration “/etc/my.cnf file” [client] socket=/mnt/volume-nyc1-01/mysql/mysql.sock 4) sudo systemctl start mariadb 5) mysql -u root -p 6) select @@datadir; 7) In output new data directory &#34;/mnt/volume-nyc1-01/mysql/&#34; should be visible </pre>kanika satijahttps://mariadb.com/kb/en/mariadb-server-wont-start-after-moving-the-data-directory/+comments/3002
