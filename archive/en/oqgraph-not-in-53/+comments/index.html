Re: OQGRAPH not in 5.3? [Answer]

The OQGraph plugin is included in MariaDB, but it is disabled by default. To use it, use INSTALL PLUGIN to activate it, like so:

MariaDB [(none)]> INSTALL PLUGIN oqgraph SONAME 'ha_oqgraph.so';
Query OK, 0 rows affected (0.48 sec)

This only needs to be done once. Once activated, OQGRAPH will appear in the list of installed storage engines:

MariaDB [(none)]> SHOW ENGINES;
+------------+---------+--------------------------------------------------------------------------------------------------+--------------+------+------------+
| Engine     | Support | Comment                                                                                          | Transactions | XA   | Savepoints |
+------------+---------+--------------------------------------------------------------------------------------------------+--------------+------+------------+
...
| OQGRAPH    | YES     | Open Query Graph Computation Engine, stored in memory (http://openquery.com/graph)               | NO           | NO   | NO         |
+------------+---------+--------------------------------------------------------------------------------------------------+--------------+------+------------+
11 rows in set (0.00 sec)

Re: OQGRAPH not in 5.3?

Be aware that if you build from source, you'll need the BOOST library in order to build OQGRAPH: http://www.boost.org.

The first time I tried to install OQGRAPH from MariaDB, it complained that it wasn't available. Then I discovered that you need to feed "--with-plugin_oqgraph" to the configure script. That didn't work either, complaining about a missing "boost" directory. That led me on a googlehunt that eventually turned up that open-source lib.

Glad it was a sunny day. If I had done all that work on a rainy day, I'd be upset at not being outside working the farm instead… :-)

I'm contacting the OQGRAPH authors to ask that they please include BOOST in their distribution, or at least mention it prominently.
