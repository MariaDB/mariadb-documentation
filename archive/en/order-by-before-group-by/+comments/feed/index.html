W.https://mariadb.com/kb/en/order-by-before-group-by/+comments/4686Re: ORDER BY before GROUP BYhttps://mariadb.com/kb/en/order-by-before-group-by/+comments/4685<p>As the <a href="/kb/en/group-by/">GROUP BY</a> article states, "If you select a non-grouped column or a value computed from a non-grouped column, it is undefined which row the returned value is taken from. This is not permitted if the <code>ONLY_FULL_GROUP_BY</code> <a href="/kb/en/sql-mode/">SQL_MODE</a> is used." So you can't guarantee the order. You however use a function such as MAX(info) to get a specific value. </p> <pre class="fixed">SELECT COUNT(name), name, MAX(info) FROM tb_user GROUP BY name ORDER BY id; +-------------+-------+-----------+ | COUNT(name) | name | MAX(info) | +-------------+-------+-----------+ | 3 | Peter | text3 | | 2 | Paul | text2 | +-------------+-------+-----------+ </pre><p>Since the text is unlikely to be sortable in that way, here's an extremely horrible hack that would work, assuming your ids increment:</p> <pre class="fixed">SELECT COUNT(name), name, MAX(CONCAT(id,': ',info)) FROM tb_user GROUP BY name ORDER BY id; +-------------+-------+---------------------------+ | COUNT(name) | name | MAX(CONCAT(id,': ',info)) | +-------------+-------+---------------------------+ | 3 | Peter | 4: text3 | | 2 | Paul | 5: text2 | +-------------+-------+---------------------------+ </pre><p>Of course, you probably want to remove the id from the info field, so taking the hack further, and assuming the id length never exceeds 50:</p> <pre class="fixed">SELECT COUNT(name), name, SUBSTR(MAX(CONCAT(LPAD(id,50),info)),51) AS info FROM tb_user GROUP BY name ORDER BY id; +-------------+-------+-------+ | COUNT(name) | name | info | +-------------+-------+-------+ | 3 | Peter | text3 | | 2 | Paul | text2 | +-------------+-------+-------+ </pre>Ian Gilfillanhttps://mariadb.com/kb/en/order-by-before-group-by/+comments/4685
