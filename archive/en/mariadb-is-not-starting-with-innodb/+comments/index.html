Re: MariaDB is not starting with InnoDB. [Answer]

InnoDB can refuse to start for various reasons. If InnoDB does not start, the MariaDB Server can start up. In that case, no InnoDB options or tables will be recognized.

The reason for InnoDB not starting up should be in the server’s error log, which could be written to a file like 'mysqld.err'. If logs are managed by systemd, then you could find https://mariadb.com/kb/en/library/systemd/ useful.

InnoDB should report a reason for it not starting up. Usually it also writes some messages to the log before crashing. Once you have read the error log, you can search https://jira.mariadb.org/ for the error messages. Finally, you could post additional information to an existing open ticket (if you found one), file a new ticket, or upgrade to a newer version that might have addressed the problem.
