The main concern with skipping versions is that, while upgrading one major version is usually well-tested, skipping versions is not, so you may bump into an incompatibility. I suggest carefully going through the changes in each release to see if any could affect you: <a href="/kb/en/what-is-mariadb-53/">5.3</a>, <a href="/kb/en/what-is-mariadb-55/">5.5</a>, <a href="/kb/en/what-is-mariadb-100/">10.0</a>, <a href="/kb/en/what-is-mariadb-101/">10.1</a> as well as <a href="/kb/en/mariadb-vs-mysql-compatibility/">MariaDB versus MySQL - Compatibility</a>. Many people have successfully done what you intend, but it depends entirely on your usage.</p> <p>2/3. InnoDB is very different to MyISAM. It's much more robust, and there's usually no need to check and repair in the same way. See <a href="/kb/en/converting-tables-from-myisam-to-innodb/">Converting Tables from MyISAM to InnoDB</a> as well as <a href="/kb/en/xtradbinnodb-recovery-modes/">XtraDB/InnoDB Recovery Modes</a> for what happens when innoDB detects corruption. Usually you would need to restore from backup (drop and reload).</p> Ian Gilfillanhttps://mariadb.com/kb/en/direct-upgrade-from-mysql-51-to-maria-db-101/+comments/2458
