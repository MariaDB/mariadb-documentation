MariaDB Knowledge Base Comments for: Wait for changes on a tablehttps://mariadb.com/kb/en/wait-for-changes-on-a-table/+comments/feed/en-usMon, 26 May 2025 12:55:10 +0000Re: Wait for changes on a tablehttps://mariadb.com/kb/en/wait-for-changes-on-a-table/+comments/6732<p>Like is mentioned in <a href="https://jira.mariadb.org/browse/MDEV-16590">MDEV-16590</a>, this can be emulated in SQL. Here's a quick little prototype of a pair of stored procedures and a table that can be used to send and wait for signals. Adding signal IDs and versions for notify_one/notify_all should be possible.</p> <pre class="fixed">delimiter $$ ; CREATE OR REPLACE PROCEDURE wait_for_signal(IN timeout INT) BEGIN INSERT INTO signal_waiters VALUES (CONNECTION_ID()); EXECUTE IMMEDIATE CONCAT('SELECT ''wait_for_signal'', SLEEP (', timeout, ')'); DELETE FROM signal_waiters WHERE id = CONNECTION_ID(); END $$ CREATE OR REPLACE PROCEDURE send_signal() BEGIN START TRANSACTION; FOR rec IN (SELECT id FROM signal_waiters FOR UPDATE) DO EXECUTE IMMEDIATE CONCAT('KILL QUERY ', rec.id); DELETE FROM signal_waiters WHERE id = rec.id; END FOR; COMMIT; END $$ delimiter ; $$ CREATE OR REPLACE TABLE signal_waiters(id BIGINT PRIMARY KEY); -- To wait for a signal CALL wait_for_signal(10); -- To send a signal CALL send_signal(); </pre>Markus Mäkelähttps://mariadb.com/kb/en/wait-for-changes-on-a-table/+comments/6732Re: Wait for changes on a tablehttps://mariadb.com/kb/en/wait-for-changes-on-a-table/+comments/6731<p>Not currently - there is feature <a href="https://jira.mariadb.org/browse/MDEV-16590">MDEV-16590</a>.</p> <p>A replication based client connector, not java connector unfortunately, with the right amount of replication filters might be able to listen for a change, but its not there currently.</p> <p>Sorry, polling it is.</p> Daniel Blackhttps://mariadb.com/kb/en/wait-for-changes-on-a-table/+comments/6731
