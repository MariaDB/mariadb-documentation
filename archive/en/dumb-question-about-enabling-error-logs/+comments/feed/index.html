MariaDB Knowledge Base Comments for: dumb question about enabling error logshttps://mariadb.com/kb/en/dumb-question-about-enabling-error-logs/+comments/feed/en-usTue, 13 Feb 2024 01:56:38 +0000Re: dumb question about enabling error logshttps://mariadb.com/kb/en/dumb-question-about-enabling-error-logs/+comments/4057<p>Errors are always written, but not always to log file. If <a href="/kb/en/server-system-variables/#log_error">log_error</a> is not enabled, errors will be logged to stderr. See <a href="/kb/en/error-log/">Error Log</a>. Note also that the error log, the <a href="/kb/en/general-query-log/">General Query Log</a> and the <a href="/kb/en/mariadb-audit-plugin/">Audit Plugin</a> are all different things.</p> <p>As to why the error log is not being written, check file permissions (though the server shouldn't start if it can't write the log file), or specify <code>log_error</code> with no location in order to write to the default location to test. Also, to help track down errors, I suggest doing one thing at a time - first get error logs working, then implement the audit plugin if you need it.</p> Ian Gilfillanhttps://mariadb.com/kb/en/dumb-question-about-enabling-error-logs/+comments/4057
