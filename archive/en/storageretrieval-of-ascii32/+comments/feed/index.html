So chr(32), a single space, gets interpreted as the empty string. See here:</p> <p> <a href="https://mariadb.com/kb/en/char/">https://mariadb.com/kb/en/char/</a></p> <p> " Trailing spaces are removed when CHAR values are retrieved"</p> <p>You can set the SQL_MODE to include PAD_CHAR_TO_FULL_LENGTH to disable this behaviour and be able to select the chr(32):</p> <pre> MariaDB&gt; set sql_mode = concat(@@sql_mode, ',PAD_CHAR_TO_FULL_LENGTH'); MariaDB&gt; create table t5 (a char(1)); MariaDB&gt; insert into t5 values ('a'), (' '); MariaDB&gt; select a, length(a), ascii(a) from t5; +------+-----------+----------+ | a | length(a) | ascii(a) | +------+-----------+----------+ | a | 1 | 97 | | | 1 | 32 | +------+-----------+----------+ </pre> Kristian Nielsenhttps://mariadb.com/kb/en/storageretrieval-of-ascii32/+comments/6467
