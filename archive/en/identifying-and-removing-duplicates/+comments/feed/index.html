This works by setting a unique index on all the identical fields, and by making use of the old ALTER TABLE copying behaviour, ignoring errors when it copies the data back, basically silently dropping duplicates. This requires the <a href="/kb/en/server-system-variables/#old_alter_table">old_alter_table</a> system variable to be temporarily set. Here's a sample:</p> <pre class="fixed">CREATE TABLE t (id INT, f1 VARCHAR(2)); INSERT INTO t VALUES (1,'a'), (2,'a'), (3,'b'), (4,'a'); SELECT * FROM t; +------+------+ | id | f1 | +------+------+ | 1 | a | | 2 | a | | 3 | b | | 4 | a | +------+------+ SET SESSION old_alter_table=1; ALTER IGNORE TABLE t ADD UNIQUE INDEX u(f1); Query OK, 4 rows affected (0.82 sec) Records: 4 Duplicates: 2 Warnings: 0 SELECT * FROM t; +------+------+ | id | f1 | +------+------+ | 1 | a | | 3 | b | +------+------+ SET SESSION old_alter_table=0; </pre><p>With your data, you would create the unique index on all the duplicated fields, so something like:</p> <pre class="fixed">ALTER TABLE t ADD UNIQUE INDEX u(f1, f2, f3...); </pre>Ian Gilfillanhttps://mariadb.com/kb/en/identifying-and-removing-duplicates/+comments/3151Re: Identifying and removing duplicateshttps://mariadb.com/kb/en/identifying-and-removing-duplicates/+comments/3150<p>Thanks for the info.
