MariaDB Knowledge Base Comments for: CHECKSUM TABLEhttps://mariadb.com/kb/en/checksum-table/+comments/feed/en-usMon, 02 Dec 2024 02:33:55 +0000Re: CHECKSUM TABLEhttps://mariadb.com/kb/en/checksum-table/+comments/6608<p>For those who wanted to get a checksum list of all tables of the current database:</p> <p>``` SET @string = NULL; SELECT GROUP_CONCAT(CONCAT("`",table_name,"`") SEPARATOR ",") INTO @string FROM information_schema.tables WHERE table_schema = DATABASE() ORDER BY table_name ASC; SET @string = CONCAT("CHECKSUM TABLE ", @string, " EXTENDED;"); PREPARE stmt FROM @string; EXECUTE stmt; DEALLOCATE PREPARE stmt; ```</p> <p>And to save this into a file:</p> <p>``` ... mysql -u username my_database_name -e ' SET @string = NULL; SELECT GROUP_CONCAT(CONCAT("`",table_name,"`") SEPARATOR ",") INTO @string FROM information_schema.tables WHERE table_schema = DATABASE() ORDER BY table_name ASC; SET @string = CONCAT("CHECKSUM TABLE ", @string, " EXTENDED;"); PREPARE stmt FROM @string; EXECUTE stmt; DEALLOCATE PREPARE stmt; ' &gt; path/to/my_checksum.log ```</p> hans wienerschnitzelhttps://mariadb.com/kb/en/checksum-table/+comments/6608Re: CHECKSUM TABLEhttps://mariadb.com/kb/en/checksum-table/+comments/2837<p>There is a typo: "however, also for identical tables there is always a very slight chance that they return the same value."</p> <p>should be</p> <p>"however, also for *non-*identical tables there is always a very slight chance that they return the same value."</p> https://mariadb.com/kb/en/checksum-table/+comments/2837
