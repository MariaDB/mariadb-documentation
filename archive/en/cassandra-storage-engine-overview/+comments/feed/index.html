I guess it will have type=ALL, which means a full table scan.</p> <p>Internally, Cassandra's Thrift API for primary key lookups is:</p> <p>list&lt;ColumnOrSuperColumn&gt; get_slice(binary key, ColumnParent column_parent, SlicePredicate predicate, ConsistencyLevel consistency_level)</p> <p>(see <a href="http://wiki.apache.org/cassandra/API10">http://wiki.apache.org/cassandra/API10</a>). Note that you have to provide one single key value, not a range of values.</p> <p>There is get_range_slices() call, which looks as if it was returning records in a range, but there is a catch. Cassandra docs say:</p> <p>"Note that when using RandomPartitioner, keys are stored in the order of their MD5 hash, making it impossible to get a meaningful range of keys between two endpoints."</p> <p>and since RandomPartitioner is the one that is typically used with Cassandra, we have to conclude that ordered scans on Primary Key are not possible in Cassandra (at least not in an efficient way). </p> Sergei Petruniahttps://mariadb.com/kb/en/cassandra-storage-engine-overview/+comments/1274Using Primary Index in Select..https://mariadb.com/kb/en/cassandra-storage-engine-overview/+comments/1273<p>I have built a simple example using a dictionary of words - with word as the primary key.</p> <p>In mariadb - if I use this query: </p> <p> select * from words where word='hello';</p> <p>1 row in set (0.07 sec)</p> <p>Works as expected - using the primary key</p> <p>then I do: select * from words where word &gt; 'hello' and word &lt; 'help'; 54 rows in set (4.62 sec)</p> <p>That is roughly the same time as a full table scan (from other experiments) so it appears not to be using the primary key in this example.</p> <p>Am I doing something wrong - or misunderstanding something fundimental...</p> <p>-Andy</p> Andrew Galewskyhttps://mariadb.com/kb/en/cassandra-storage-engine-overview/+comments/1273Re: Cassandra storage engine overview-datatypes mappinghttps://mariadb.com/kb/en/cassandra-storage-engine-overview/+comments/1241<p>Hi,</p> <p>I create a "opdscreen" table into the Cassandra.
