It seems that often, other software expects to see it in the /var/run/mysqld folder.</p> <p>The solution is to either change the setting in the my.cnf file</p> <p>socket = /tmp/mysql.sock</p> <p>or to create the required symlink.</p> <p>ln -s /tmp/mysql.sock /var/run/mysqld/mysql.sock</p> <p>[you will need to be root, so use sudo on debian based systems]</p> <p>I use the second solution as it enables you to be place a link to the file wherever a bit of software is expecting it to be.</p> David Myershttps://mariadb.com/kb/en/missing-mysqldsock-file/+comments/165Re: Missing mysqld.sock filehttps://mariadb.com/kb/en/missing-mysqldsock-file/+comments/111<p>Try doing:</p> <p>cd /etc/init.d Go to where mysqld is, and try: ./mysqld start</p> <p>Then do mysql -uroot and see if you can login.</p> <p>Also, it would help if you tell us how you installed MariaDB.
