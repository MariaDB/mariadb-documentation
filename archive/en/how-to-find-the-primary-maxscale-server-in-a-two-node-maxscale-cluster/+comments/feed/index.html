MariaDB Knowledge Base Comments for: How to find the Primary Maxscale server in a two node maxscale clusterhttps://mariadb.com/kb/en/how-to-find-the-primary-maxscale-server-in-a-two-node-maxscale-cluster/+comments/feed/en-usMon, 19 May 2025 11:32:08 +0000Re: How to find the Primary Maxscale server in a two node maxscale clusterhttps://mariadb.com/kb/en/how-to-find-the-primary-maxscale-server-in-a-two-node-maxscale-cluster/+comments/7073<p>What the "primary" MaxScale means depends on your configuration.</p> <p>If you are using cooperative monitoring with the <code>mariadbmon</code> module, the output of the <code>maxctrl show monitor</code> command will contain <code>"primary": true</code> if the monitor "owns" the cluster and thus failover is controlled by this MaxScale instance.</p> <p>If you are using the legacy <code>passive</code> parameter, you can see the value of it in the <code>maxctrl show maxscale</code> output.</p> <p>If you are interested in programmatically getting the value, the <code>maxctrl api get</code> command can be useful. You can use it to navigate directly to an API endpoint and extract values from it. For example, <code>maxctrl api get monitors data[].attributes.monitor_diagnostics.primary</code> would get you the primary attribute of each monitor and <code>maxctrl api get monitors/&lt;name-of-monitor&gt; data.attributes.monitor_diagnostics.primary</code> would get you the primary attribute of a specific monitor.</p> <p>The REST-API tutorial will tell you how to enable HTTPS for the GUI: <a href="https://mariadb.com/kb/en/mariadb-maxscale-2402-maxscale-2402-rest-api-tutorial/">https://mariadb.com/kb/en/mariadb-maxscale-2402-maxscale-2402-rest-api-tutorial/</a></p> Markus Mäkelähttps://mariadb.com/kb/en/how-to-find-the-primary-maxscale-server-in-a-two-node-maxscale-cluster/+comments/7073
