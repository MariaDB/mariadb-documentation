You need to either remove them at all, or to replace them with back ticks:</p> <pre> create table if not exists addresslist( `id` int not null auto_increment, `firstname` varchar not null, `lastname` varchar not null, `city` varchar not null, `state` char(2), `country` text default 'usa', `zip` text(10) not null primary key (id) ); </pre> <p>Then you will start getting another syntax error:</p> <pre> near 'not null, `lastname` varchar not null, `city` varchar not null, `state` char(' </pre> <p>It will be happening because VARCHAR requires a length parameter, so you will need to replace all varchar with something like varchar(64), or whatever length you need:</p> <pre> create table if not exists addresslist( `id` int not null auto_increment, `firstname` varchar(64) not null, `lastname` varchar(64) not null, `city` varchar(64) not null, `state` char(2), `country` text default 'usa', `zip` text(10) not null primary key (id) ); </pre> <p>Then there will be error </p> <pre> BLOB/TEXT column 'country' can't have a default value </pre> <p>which is rather self-explanatory: you need to remove the default value from the text column:</p> <pre> create table if not exists addresslist( `id` int not null auto_increment, `firstname` varchar(64) not null, `lastname` varchar(64) not null, `city` varchar(64) not null, `state` char(2), `country` text, `zip` text(10) not null primary key (id) ); </pre> <p>Now you will have a syntax error again:</p> <pre> near '(id) )' </pre> <p>which is caused by a missing comma before primary key definition. Fix it, and your query will work:</p> <pre> &gt; create table if not exists addresslist( -&gt; `id` int not null auto_increment, -&gt; `firstname` varchar(64) not null, -&gt; `lastname` varchar(64) not null, -&gt; `city` varchar(64) not null, -&gt; `state` char(2), -&gt; `country` text, -&gt; `zip` text(10) not null, -&gt; primary key (id) ); Query OK, 0 rows affected (0.03 sec) </pre> <p>P.S.
