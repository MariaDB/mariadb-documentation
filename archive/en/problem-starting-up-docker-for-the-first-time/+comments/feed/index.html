The common bit is the <code> -x "$dirname0/@bindir@/my_print_defaults"</code> part of <code>my_print_defaults</code> failing the same way.</p> <p>In short the <code>docker</code> version you are using in the (end of life) squeeze is filtering out the faccess2 system call that bash is using to test the presence of <code>my_print_defaults</code>.</p> <p>--security-opt seccomp=unconfined might help with your current version, but a better solution is upgrading squeeze and the OCI container runtime that comes with it. </p> <p>I also recommend <code>podman</code> instead of <code>docker</code>. The command line options are the same. . Debian unstable versions have <code>podman-docker</code>.
