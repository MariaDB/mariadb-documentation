MariaDB Knowledge Base Comments for: What are these files written in temporary folder /tmp/MYXFhjiUhttps://mariadb.com/kb/en/what-are-these-files-written-in-temporary-folder-tmpmyxfhjiu/+comments/feed/en-usMon, 16 Oct 2023 23:10:00 +0000Re: What are these files written in temporary folder /tmp/MYXFhjiUhttps://mariadb.com/kb/en/what-are-these-files-written-in-temporary-folder-tmpmyxfhjiu/+comments/5489<p>Temporary files without a name on the filesystem is standard way to use them for temporary tables, alter tables and other temporary uses. It means if that MariaDB is suddenly stopped there is no cleanup to do. They might be big, but there's nothing to suggest they won't finish eventually.</p> <p>To find out the cause look at the <a href="/kb/en/show-processlist/">SHOW PROCESSLIST</a> for the ones running a long time (they might have a status that says something about temporary tables). Use <a href="/kb/en/show-explain/">SHOW EXPLAIN FOR {threadid}</a> so see what they are doing. Look for temporary table activity.</p> <p>Given this is occurring after a bunk load. Make its the query plan statistics that are confused/lost and sub-optimal queries requiring temporary tables are occuring. Try an <a href="/kb/en/analyze-table/">ANALYZE TABLE {table}</a> as part of the import process.</p> Daniel Blackhttps://mariadb.com/kb/en/what-are-these-files-written-in-temporary-folder-tmpmyxfhjiu/+comments/5489
