After all, it will use other tables than the rest of the applications so there shouldn't be any conflicting locks. In the worst case you can fire up another independent MariaDB server and have zabbix write it's results there.</p> <p>Any system monitoring tool that monitors disk and memory should help you see the pressure on the system.
