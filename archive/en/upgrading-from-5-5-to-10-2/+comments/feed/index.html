MariaDB Knowledge Base Comments for: Upgrading from 5.5 to 10.2https://mariadb.com/kb/en/upgrading-from-5-5-to-10-2/+comments/feed/en-usMon, 14 Apr 2025 13:16:47 +0000Re: Upgrading from 5.5 to 10.2https://mariadb.com/kb/en/upgrading-from-5-5-to-10-2/+comments/5827<p>There are <a href="https://mariadb.org/download/?t=repo-config&amp;d=Red+Hat+EL+7+%28x86_64%29&amp;v=10.2&amp;r_m=xtom_tal">repositories for 10.2 for RHEL7</a> that can be used to install the MariaDB-10.2 code.</p> <p>Copy the datadir (/var/lib/mysql by default) to your new server. If there is any /etc/my.cnf or /etc/my.cnf.d files on the previous Cloudera release copy these over to the new server too if they container non-default values. Some options may have changed, but the error log on startup will clearly show this if it has changed. There are some <a href="/kb/en/upgrading/">documents related to upgrading</a> to read though what has changed in the major versions.</p> <p>No intermediate upgrade version is required.</p> Daniel Blackhttps://mariadb.com/kb/en/upgrading-from-5-5-to-10-2/+comments/5827
