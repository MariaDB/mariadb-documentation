When the client closes the connection, the file will be closed. Each session will have a unique file name where the value of the <code>filebase</code> is appended with the session ID.</p> <p>In 2.1, you can use the <code>log_type=unified</code> parameter to log all sessions into one file. This should immediately create the file in the specified directory.</p> Markus Mäkelähttps://mariadb.com/kb/en/why-the-filebase-option-is-not-working-on-etcmaxscalecnf/+comments/3239
