MariaDB Knowledge Base Comments for: MariaDB 10.0 installation errors on Linux Mint 16 https://mariadb.com/kb/en/mariadb-100-installation-errors-on-linux-mint-16/+comments/feed/en-usTue, 27 Feb 2024 02:56:51 +0000Re: MariaDB 10.0 installation errors on Linux Mint 16 https://mariadb.com/kb/en/mariadb-100-installation-errors-on-linux-mint-16/+comments/1182<p>Hi, </p> <p>There is not enough information to say what can be wrong on your system. I installed Saucy packages on Mint 15, which is closest to 16 that I have at hand, they got installed all right, so it's not the packages. Most likely you have some remains of previous installations on your machine. Also, it is unclear how exactly you are installing the packages -- are you running <code>apt-get</code> from the command line, or doing upgrade, or using some automatic installation tool, or ...?</p> <p>Please first try to run <code>apt-get install -f</code>. However strange it sounds, try to run it two or three times, it is known to work on the 2nd attempt sometimes. </p> <p>If it doesn't help, run <code>dpkg -l | grep -i 'maria\|mysql'</code> and see what packages you have there. Remove old installations of MariaDB or MySQL or Percona server, and, if the old data is of no essence, purge everything and then do clean install. </p> <p>If it still does not help, check the syslog or, if it exists, the server error log in <code>/var/lib/mysql</code>, it is possible that your server does not start and that's what causes the problem.</p> Elena Stepanovahttps://mariadb.com/kb/en/mariadb-100-installation-errors-on-linux-mint-16/+comments/1182
