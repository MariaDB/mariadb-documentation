Packages from MariaDB repository install mysql.service, both for 5.5 and 10.0. Packages from Fedora repository install mysqld.service and its alias mariadb.service; but there is only <a href="/kb/en/what-is-mariadb-55/">MariaDB 5.5</a> available in the Fedora repo at the moment. </p> <p>So, MariaDB programmers did not really remove anything service-related, but you would have mariadb.service deleted if you had mariadb-server (5.5) installed from the Fedora repo, and now installed 10.0 instead. </p> Elena Stepanovahttps://mariadb.com/kb/en/mariadb-community-upgraded-to-mariadb-100-but-the-systemctl-status-mariadbs/+comments/1161Re: upgraded to MariaDB 10.0 but the systemctl status mariadb.service is not working? mysql command works instead?https://mariadb.com/kb/en/mariadb-community-upgraded-to-mariadb-100-but-the-systemctl-status-mariadbs/+comments/1160<p>&lt;html&gt;</p> <p>&lt;br&gt; systemctl status mysql.service =l</p> <p>&lt;br&gt; mysql.service - LSB: start and stop MySQL &lt;br&gt; Loaded: loaded (/etc/rc.d/init.d/mysql) &lt;br&gt; </p> <p>Active: active (running) since Fri 2014-04-04 05:56:31 CDT; 1 day 14h ago &lt;br&gt; CGroup: /system.slice/mysql.service&lt;br&gt; ├─1036 /bin/sh /usr/bin/mysqld_safe --datadir=/var/lib/mysql --pid- file=/var/lib/mysql.pid&lt;br&gt; └─1191 /usr/sbin/mysqld --basedir=/usr --datadir=/var/lib/mysql --&lt;br&gt; plugin-dir=/usr/lib64/mysql/plugin --user=mysql --log-&lt;br&gt; error=/var/lib/mysql/.err --pid-file=/var/lib/mysql/.&lt;br&gt; </p> <p>Apr 04 05:56:29 systemd[1]: Starting LSB: start and stop MySQL...&lt;br&gt; Apr 04 05:56:31 mysql[1017]: Starting MySQL..[ OK ]&lt;br&gt; Apr 04 05:56:31 systemd[1]: Started LSB: start and stop MySQL.&lt;br&gt; &lt;br&gt; \x3dl.service&lt;br&gt; Loaded: not-found (Reason: No such file or directory)&lt;br&gt; &lt;br&gt; Active: inactive (dead)&lt;br&gt; &lt;/html&gt;</p> Cyberdome Cyberdomehttps://mariadb.com/kb/en/mariadb-community-upgraded-to-mariadb-100-but-the-systemctl-status-mariadbs/+comments/1160
