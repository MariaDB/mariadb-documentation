Default collations are hard-coded. For example, utf8_general_ci is the hard-coded default collation for the character set utf8.</p> <p>This is a missing feature.</p> <p>The SQL standard has "<code>CREATE CHARACTER SET</code>" statement for this: <a href="https://kb.askmonty.org/en/create-character-set-statement/">https://kb.askmonty.org/en/create-character-set-statement/</a></p> <p>So one can do:</p> <pre class="fixed"><span class="k">CREATE</span> <span class="nb">CHARACTER</span> <span class="k">SET</span> <span class="n">utf8my</span> <span class="k">AS</span> <span class="k">GET</span> <span class="n">utf8</span> <span class="k">COLLATE</span> <span class="n">utf8_unicode_ci</span><span class="p">;</span> </pre><p>and then use this:</p> <pre class="fixed"><span class="k">SET</span> <span class="k">NAMES</span> <span class="n">utf8my</span><span class="p">;</span> </pre><p>which effectively sets the connection character set to utf8, and the collation to utf8_unicode_ci.</p> <p>MariaDB does not support <code>CREATE CHARACTER SET</code> yet.</p> <p>The only workaround is to make the application send:</p> <pre class="fixed"><span class="k">SET</span> <span class="k">NAMES</span> <span class="n">utf8</span> <span class="k">COLLATE</span> <span class="n">utf8_unicode_ci</span><span class="p">;</span> </pre><p>instead of just:</p> <pre class="fixed"><span class="k">SET</span> <span class="k">NAMES</span> <span class="n">utf8</span><span class="p">;</span> </pre>Alexander Barkovhttps://mariadb.com/kb/en/change-is_default-value-for-information_schemacollations/+comments/564Re: Change IS_DEFAULT value for INFORMATION_SCHEMA.COLLATIONShttps://mariadb.com/kb/en/change-is_default-value-for-information_schemacollations/+comments/551<p>Thanks for your reply but this does not really answer my question.</p> <p>To be more precise: my question is: How can I influence the behavior of the following command:</p> <pre class="fixed">SET NAMES utf8;
