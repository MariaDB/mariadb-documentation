Against MySQL client library from Oracle? That would explain the error you're seeing.</p> <p>When we added symbol versioning to our client library we carefully tried to keep it drop-in replacement and picked exactly the same versioning that CentOS mysql-libs package uses.</p> <p>In this versioning logic, mysql_options has version libmysqlclient_16, because, indeed, it was present in the old libmysqlclient.so.16 and it haven't changed since. Thus it retained the old version.</p> <p>MySQL client library from Oracle, on the other hand, has all symbols under the new libmysqlclient_18 version, even symbols that have not changed when the version was incremented. This makes Oracle libmysqlclient.so.18 incompatible with CentOS mysql-libs and with our client library.</p> <p>I would recommend you to recompile pdo_mysql.so using either CentOS or our libmysqlclient.so</p> Sergei Golubchikhttps://mariadb.com/kb/en/unable-to-get-pdo_mysql-to-work-with-centos-6-and-mariadb-5532/+comments/814
