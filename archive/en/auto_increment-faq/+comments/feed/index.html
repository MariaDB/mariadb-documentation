MariaDB Knowledge Base Comments for: AUTO_INCREMENT FAQhttps://mariadb.com/kb/en/auto_increment-faq/+comments/feed/en-usMon, 16 Sep 2024 03:58:07 +0000Re: AUTO_INCREMENT FAQhttps://mariadb.com/kb/en/auto_increment-faq/+comments/4958<p>Hello, Regarding section "Does the auto_increment field have to be primary key?" I would strongly advise to say YES (or for MyISAM the auto_increment key must BE PART OF the primary key).</p> <p>I had a very hard time with a MyISAM table with an auto_increment field that was declared as a simple KEY : the replication kept on failing with errors message like "Ca'nt find record in mytable":</p> <p>Slave SQL: Could not execute Update_rows_v1 event on table mydb.mytable ; Can't find record in 'mytable', Error_code: 1032; handler error HA_ERR_END_OF_FILE; the event's master log mysql-bin...., end_log_pos ..., Gtid 0-1-..., Internal MariaDB error code: 1032</p> <p>We altered the MyISAM table to set the auto_increment field as the primary key and the replication worked fine afterwards...</p> <p>Regards,</p> <p>E.</p> Emmanuel KARTMANNhttps://mariadb.com/kb/en/auto_increment-faq/+comments/4958
