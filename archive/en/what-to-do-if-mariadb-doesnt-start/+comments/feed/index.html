Save the file</p> <p>3. sudo systemd daemon-reload # refresh the systemd services so it sees your override file</p> <p>4. sudo systemctl start mysql # and it should finally start</p> <p>I spent ages searching for AppArmor and SELinux fixes to this problem, when it's just systemd preventing MySQL from writing to /home.</p> Damien Norrishttps://mariadb.com/kb/en/what-to-do-if-mariadb-doesnt-start/+comments/3018Re: What to Do if MariaDB Doesn't Starthttps://mariadb.com/kb/en/what-to-do-if-mariadb-doesnt-start/+comments/2968<p>As Leho mentioned it may be also a case, f.e. if you update your Debian 8 installation to Debian 9.
