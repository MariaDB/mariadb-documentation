MariaDB Knowledge Base Comments for: Manual SST of Galera Cluster Node With Mariabackuphttps://mariadb.com/kb/en/manual-sst-of-galera-cluster-node-with-mariabackup/+comments/feed/en-usMon, 18 Mar 2024 20:31:13 +0000Re: Manual SST of Galera Cluster Node With Mariabackuphttps://mariadb.com/kb/en/manual-sst-of-galera-cluster-node-with-mariabackup/+comments/5031<p>I prefer to use pure scp, it is faster and straightforward, no need to play with grastate.dat. Something like this:</p> <p>Here we assume that we have one node (node1) running, started with galera_new_cluster. All other nodes are stopped. And MaxScales stopped</p> <p>Perform this on all stopped nodes 2-5: Wipe datadirs on those stopped nodes (nodes 2-5), NOT on that running node! (node1). Also on slave if existing. </p> <pre class="fixed">find /var/lib/mysql -type f -not -path &#34;/var/lib/mysql/example/*&#34; -delete </pre><p>Perform this on all stopped nodes 2-5: Give permissions for your account to /var/lib/mysql. This is needed because root cannot scp files between the nodes. Use id &lt;loginname&gt; too see your group. Also on slave if existing.</p> <pre class="fixed">chown -R myid:mygroup /var/lib/mysql/ </pre><p>Stop running node (node1) </p> <pre class="fixed">systemctl stop mariadb </pre><p>Start screens as user root for the scp’s on that node (node1) which was running Galera. You may want to utilise 4 terminals.</p> <pre class="fixed">screen -S fornode2 screen -S fornode3 screen -S fornode4 screen -S fornode5 screen -S fornode6 (for possible existing slave) </pre><p>And on every screen:</p> <pre class="fixed">scp -pr /var/lib/mysql/* myid@node2IP:/var/lib/mysql/ &amp;&amp; touch /tmp/scp_transfer_done_node2 </pre><pre class="fixed">scp -pr /var/lib/mysql/* myid@node3IP:/var/lib/mysql/ &amp;&amp; touch /tmp/scp_transfer_done_node3 </pre><pre class="fixed">scp -pr /var/lib/mysql/* myid@node4IP:/var/lib/mysql/ &amp;&amp; touch /tmp/scp_transfer_done_node4 </pre><pre class="fixed">scp -pr /var/lib/mysql/* myid@node5IP:/var/lib/mysql/ &amp;&amp; touch /tmp/scp_transfer_done_node5 </pre><pre class="fixed">scp -pr /var/lib/mysql/* myid@node6IP:/var/lib/mysql/ &amp;&amp; touch /tmp/scp_transfer_done_node6 </pre><p>Wait until scp’s are finished. (watch -n 30 -d /tmp/)</p> <p>Clear logs on node 2-5 if writing logs to /var/lib/mysql. Also on slave if existing</p> <pre class="fixed"> :&gt; /var/lib/mysql/mariadb.err /var/lib/mysql/slow-queries.log </pre><p>Set permissions back for mysql on nodes 2-5, also on slave if existing</p> <pre class="fixed">chown -R mysql:mysql /var/lib/mysql/ </pre><p>Bootstrap Galera Cluster from node (node1) </p> <pre class="fixed">galera_new_cluster &amp; tail -f /var/lib/mysql/mariadb.err </pre><p>Start other nodes (2-5) one-by-one. Wait that node has joined to the cluster with IST before proceeding to start next node.</p> <pre class="fixed">systemctl start mariadb &amp; tail -f /var/lib/mysql/mariadb.err </pre><p>Start MaxScales Start replication</p> Mika Hakalahttps://mariadb.com/kb/en/manual-sst-of-galera-cluster-node-with-mariabackup/+comments/5031
