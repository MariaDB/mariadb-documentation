Then all database grants can be placed on the role and equally apply to the foo@% and foo@localhost users.</p> <pre class="fixed">MariaDB [(none)]&gt; create user foo identified by 'foo'; Query OK, 0 rows affected (0.001 sec) MariaDB [(none)]&gt; create user foo@localhost identified by 'foo'; Query OK, 0 rows affected (0.001 sec) MariaDB [(none)]&gt; create role foo_role; Query OK, 0 rows affected (0.001 sec) MariaDB [(none)]&gt; grant foo_role to foo; Query OK, 0 rows affected (0.001 sec) MariaDB [(none)]&gt; grant foo_role to foo@localhost; Query OK, 0 rows affected (0.001 sec) MariaDB [(none)]&gt; create database fdatabase; Query OK, 1 row affected (0.001 sec) MariaDB [(none)]&gt; grant all on fdatabase.* to foo_role; Query OK, 0 rows affected (0.001 sec) </pre>Daniel Blackhttps://mariadb.com/kb/en/grants-from-host-vs-localhost/+comments/5370Re: Grants from host % vs localhosthttps://mariadb.com/kb/en/grants-from-host-vs-localhost/+comments/5369<p>Hi, </p> <p>It would be much easier if you could pass the links and provide an example what you would like to achieve, but let me see if I understood you correctly.<br> When you grant privileges to the user, you can specify to which <em>user@host</em> pair it should apply.<br> Let's look an example.
