MariaDB Knowledge Base Comments for: Certificate Creation with OpenSSLhttps://mariadb.com/kb/en/certificate-creation-with-openssl/+comments/feed/en-usMon, 30 Dec 2024 04:23:44 +0000Re: Certificate Creation with OpenSSLhttps://mariadb.com/kb/en/certificate-creation-with-openssl/+comments/4242<p>Creating the private certificate with <code>openssl req -newkey [...]</code> does create the file in the PKCS#8 format (at least since Debian Wheezy) which is not recognised by YaSSL.</p> <p><a href="/kb/en/what-is-mariadb-101/">MariaDB 10.1</a> shipped with Debian Stretch is linked with YaSSL and thus cannot load the certificate (leading to a <code>SSL error: Unable to get private key from '/path/to/key.pem'</code> error and SSL not working) but <a href="/kb/en/what-is-mariadb-101/">MariaDB 10.1</a> from the MariaDB.org repository is linked with OpenSSL and does load the certificate without any issue.</p> <p>The solution is to convert the certificate from PKCS#8 to PKCS#1 using such a command : <code>openssl rsa -in pkcs8.key -out pkcs1.key</code></p> <p>Or to use a MariaDB binary that is linked to OpenSSL and not YaSSL.</p> <p>More infos on <a href="https://bugs.debian.org/cgi-bin/bugreport.cgi?bug=927289">https://bugs.debian.org/cgi-bin/bugreport.cgi?bug=927289</a></p> <hr> <p>ps: PKCS#1 files starts with <code>-----BEGIN RSA PRIVATE KEY-----</code> while PKCS#8 starts with <code>-----BEGIN PRIVATE KEY-----</code></p> Jean Weisbuchhttps://mariadb.com/kb/en/certificate-creation-with-openssl/+comments/4242
