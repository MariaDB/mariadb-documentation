MariaDB Enterprise Server 10.0.20 Release Notes

This version of MariaDB Enterprise Server is built off MariaDB 10.0.20. We are now providing only 64-bit x86 (x86-64) Linux packages for MariaDB Enterprise Server; 32-bit x86 Linux packages have been discontinued. We have ended support for Fedora 19, Fedora 20, OpenSuse 13.1 and Ubuntu 10.04 LTS (Lucid) packages.

Notable Changes

  • XtraDB updated to XtraDB-5.6.24-72.2
  • Innodb updated to InnoDB-5.6.25
  • Performance Schema updated to 5.6.25
  • TokuDB updated to TokuDB-7.5.7
  • Client command line option --ssl-verify-server-cert (and MYSQL_OPT_SSL_VERIFY_SERVER_CERT option of the client API) when used together with --ssl will ensure that the established connection is SSL-encrypted and the MariaDB server has a valid certificate. This fixes CVE-2015-3152.

MariaDB Enterprise Cluster 10.0.20 Release Notes

This version of MariaDB Enterprise Cluster is built off MariaDB Galera Cluster 10.0.20 and is mainly an upgrade to benefit from the many fixes and improvements in MariaDB Galera Cluster 10.0.20, including some backported fixes, as detailed in the changelog.

We are now providing only x86-64 Linux packages for MariaDB Enterprise Cluster; 32-bit x86 Linux packages have been discontinued. We have also ended support for Fedora 19, Fedora 20, OpenSuse 13.1 and Ubuntu 10.04 LTS (Lucid) packages.

Updates and fixes in this version

  • This release is mainly a bug-fix release.
  • Codership changes: github.com/codership/mysql-wsrep/tree/5.6 (till commit 2bb4c3d).
  • The Galera library used by MariaDB Galera Cluster and included in the MariaDB repositories is currently at version 25.3.9.

Noteable Changes

  • Running MariaDB Galera Cluster 5.5 and 10.0 nodes in a cluster is not supported (MDEV-6257)
  • This version of MariaDB Galera Cluster supports wsrep API v25 which means MariaDB Galera Cluster can be used with either a 25.2.x or 25.3.x Galera wsrep provider. A 25.3.x wsrep provider is included in the MariaDB repositories.
  • On Ubuntu and Debian, the Galera Arbitrator daemon (garbd) and the galera library are in two separate packages. The packages are named galera-3 and galera-arbitrator-3. When installing MariaDB Galera Cluster on Ubuntu and Debian (the mariadb-galera-server package), the galera-3 package will be automatically installed. You can then install the arbitrator package on a separate node (recommended) or on one of the nodes running mariadb-galera-server (not recommended).

Note: If Galera 25.2.x and 25.3.x are both being used in the cluster, MariaDB with Galera 25.3.x must be started with wsrep_provider_options='socket.checksum=1' in order to make it backward compatible with Galera v2. Galera wsrep providers other than 25.3.x or 25.2.x are not supported.
