All these years later and I finally figured out how to fix this problem! Though I don't understand WHY it works. </p> <p>You have to add "sudo" to the launchd script! And that's it, it launches the database every time after that. But because it's a launchdeamon, it should already be running the rest of the command as root. So adding sudo should do absolutely nothing. And yet, it totally fixes it!! Go figure! </p> <p>Note it's still in the LaunchDeamons folder, I didn't move it to LaunchAgents and I didn't add that odd LimitLoadToSessionType key. *shrug*</p> <p>&lt;key&gt;ProgramArguments&lt;/key&gt; &lt;array&gt; &lt;string&gt;/usr/bin/sudo&lt;/string&gt; &lt;string&gt;/usr/local/opt/mariadb/bin/mariadbd-safe&lt;/string&gt; &lt;string&gt;--datadir=/usr/local/var/mysql&lt;/string&gt; &lt;/array&gt;</p> <p>This command should already being run as root, so the sudo part should not do anything.
