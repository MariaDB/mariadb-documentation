MariaDB Knowledge Base Comments for: Rotating Logs on Unix and Linuxhttps://mariadb.com/kb/en/rotating-logs-on-unix-and-linux/+comments/feed/en-usMon, 30 Sep 2024 03:44:05 +0000Re: Rotating Logs on Unix and Linuxhttps://mariadb.com/kb/en/rotating-logs-on-unix-and-linux/+comments/4333<p>First off, this a useful resource. Probably the best tutorial for Logrotate in general. Very efficient and used what was shared to tweak other log file rotations.</p> <p>Unfortunately, one error breaks the rotation of logs with Logrotate, and I ran into a few of them, but I was able to solve them quickly.</p> <hr> <p>The following is how I got this to run without errors on Centos 7 with a "mariadb" config file located in: <strong>/etc/logrotate.d/mariadb</strong>:</p> <hr> <hr> <pre class="fixed">/var/log/mysql/* { missingok create 660 mysql mysql notifempty daily minsize 1M maxsize 100M rotate 30 dateext dateformat .%Y-%m-%d compress delaycompress sharedscripts olddir archive/ createolddir 770 mysql mysql postrotate # just if mysqld is really running if test -x /usr/bin/mysqladmin &amp;&amp; \ /usr/bin/mysqladmin ping &amp;&gt;/dev/null then /usr/bin/mysqladmin --local flush-error-log \ flush-engine-log flush-general-log flush-slow-log fi endscript su mysql mysql } </pre><hr> <hr> <p>Here the errors I was encountering when using the shared command above verbatim. </p> <hr> <p><strong>error: /etc/logrotate.d/mariadb:1 unknown option 'sudo' -- ignoring line</strong></p> <p>Since my config file went in /etc/logrotate.d/mariadb, I just removed "sudo tee /etc/logrotate.d/mariadb" and since the &lt;&lt;EOF operator was not reading throughout to the delimiter, I removed all references to "end of file". I am not sure if this is the right situation to use EOD in bash, but I believe they would have to be on inside the curly brackets (or possible outside as an option) to function.</p> <hr> <p><strong>error: /etc/logrotate.d/mariadb:7 bad size '1M # only use with logrotate &gt;= 3.7.4'</strong></p> <p><strong>error: /etc/logrotate.d/mariadb:7 bad size '100M # only use with logrotate &gt;= 3.8.1'</strong></p> <p>The commented out notes after the commands were being included within the command, so I removed:</p> <pre class="fixed"># only use with logrotate &gt;= 3.7.4 # only use with logrotate &gt;= 3.8.1 # only use if your logrotate version is compatible with below dateformat </pre><hr> <p><strong>error: skipping "/var/log/mysql/mariadb.err" because parent directory has insecure permissions (It's world writable or writable by group which is not "root") Set "su" directive in config file to tell logrotate which user/group should be used for rotation.</strong></p> <p><strong>error: skipping "/var/log/mysql/mariadb.log" because parent directory has insecure permissions (It's world writable or writable by group which is not "root") Set "su" directive in config file to tell logrotate which user/group should be used for rotation.</strong></p> <p><strong>error: skipping "/var/log/mysql/mariadb-slow.log" because parent directory has insecure permissions (It's world writable or writable by group which is not "root") Set "su" directive in config file to tell logrotate which user/group should be used for rotation.</strong></p> <p>To fix the above permissions errors, I added <em><strong>su mysql mysql</strong></em> after the "endscript" to tell logrotate which user/group should be used for rotation.</p> <hr> <p><strong>error: found error in /var/log/mysql/* , skipping</strong></p> <p>Letting you know it's going to be skipping this rotation command. </p> <hr> <p>Tiny's comment was spot on. The trailing "}" is also missing. It took me a second glance when I first reviewed this article, but this stood out pretty clear right off the top.</p> <hr> <p>Strange, because in <strong>/var/lib/logrotate/logrotate.status</strong> the date and time showed up perfect with:</p> <p><code class="fixed" style="white-space:pre-wrap">dateformat.%Y-%m-%d-%H-%M-%S</code></p> <p>But the archive names came out as:</p> <p><strong>mariadb.log.2020-01-04-10-%M-%S</strong></p> <p>So, I choose to slim it down to:</p> <p><code class="fixed" style="white-space:pre-wrap">dateformat .%Y-%m-%d</code></p> <hr> <p>I hope this helps anyone who is looking for it.</p> Streamer Cloudhttps://mariadb.com/kb/en/rotating-logs-on-unix-and-linux/+comments/4333Tiny Logrotate Omissionhttps://mariadb.com/kb/en/rotating-logs-on-unix-and-linux/+comments/4172<p>There's a tiny mistake in the logrotate entry -- there needs to be a closing brace (}) before the EOF.</p> Jeff Williamshttps://mariadb.com/kb/en/rotating-logs-on-unix-and-linux/+comments/4172
