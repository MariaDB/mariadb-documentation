MariaDB Knowledge Base Comments for: Dynamic Columns APIhttps://mariadb.com/kb/en/dynamic-columns-api/+comments/feed/en-usMon, 10 Jul 2023 19:52:09 +0000Re: Dynamic Columns APIhttps://mariadb.com/kb/en/dynamic-columns-api/+comments/864<p>Currently, there are bunch of issues with this API on the client side. To begin wirh, to compile with ma_dyncol.h, you first need to include my_global.h, my_sys.h, m_string.h. This would be OK if it wasn't for the fact that my_global.h in turns needs my_byteorder.h, which is not part of the binary distribution (at least not the tarball), it's only there in the source. The same is tru for my_byteorder.h and this in turn includes little_endian.h or big_endian.h, which also do not exists.</p> <p>So having managed to get the includefiles right, there are some more issues like mariadb_dyncol_init() isn't defined. I can't determine from the doc above if it should be or not, but in any case I think it should really be included. mariadb_dyncol_free doesn't exists and isn't defined as a macro, instead you can use dynamic_column_column_free() which is defined as dynstr_free(V).</p> <p>As for ma_dyncol.h, this lacks a:</p> <ol><li>ifdef <u>cplusplus extern "C" {</u> </li><li>endif ... </li><li>ifdef <u>cplusplus }</u> </li><li>endif </li></ol> <p>So it will not compile correctly with C++.</p> <p>All this tested on Linux (Ubuntu) x86_64</p> Anders Karlssonhttps://mariadb.com/kb/en/dynamic-columns-api/+comments/864
