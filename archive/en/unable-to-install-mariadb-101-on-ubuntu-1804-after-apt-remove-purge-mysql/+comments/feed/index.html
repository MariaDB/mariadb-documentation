So I backed out and reinstalled so I could re-enter the root password again. What I did not understand, since this is the first time in 7 years that I have installed either MySQL or MariaDB from scratch as a result of a new OS install, is that the behavior of the root account has changed in the past 7 years. The change was made for excellent reasons but the documentation of the new behavior was buried somewhere in 7 years of accumulated documentation. In old releases one accessed the server by the command "mysql -u root -p", but that has been changed to the more secure "sudo mysql -u root". Because the error message that the mysql client returns is the same one issued for bad password this change in behavior was astonishing. It took me two days of desperately uninstalling and re-installing MariaDB before I found the documentation. Also the installation problem caused ALL apt get install requests to fail because the dpkg update command always failed because of the incomplete installation of MariaDB.</p> <p>I have everything up and running now but this was a painful experience which I know was unintended.</p> James Cobbanhttps://mariadb.com/kb/en/unable-to-install-mariadb-101-on-ubuntu-1804-after-apt-remove-purge-mysql/+comments/3402
