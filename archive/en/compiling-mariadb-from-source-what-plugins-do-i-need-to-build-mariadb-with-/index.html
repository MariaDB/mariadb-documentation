What plugins do I need to build MariaDB with to have support for MaxScale for high availability (HA)?

I want to build MariaDB from source but also ensure I am able to use something like MaxScale for high availability (HA) in my Debian system. Do I need to include any specific plugins for that? I want to make my MariaDB servers able to be replicated in a cluster (Master/Slave servers).

I saw a video about MaxScale on the official MariaDB YouTube channel: https://www.youtube.com/watch?v=mgFaKAwqaBM

Will this work for me if I build MariaDB using the following commands?

sudo apt update
sudo apt build-dep mariadb-server
cd ~
wget https://archive.mariadb.org/mariadb-11.7.2/source/mariadb-11.7.2.tar.gz
tar -xf mariadb-*.tar.gz
cd mariadb-11.7.2

cmake . -DWITHOUT_DYNAMIC_PLUGINS=1 -DPLUGIN_PERFSCHEMA=NO -DWITH_UNIT_TESTS=0 -DWITH_MARIABACKUP=ON

cmake --build . -j $(nproc)

sudo cmake --install .

sudo adduser --system --no-create-home --shell /bin/false --disabled-login --group mysql
sudo chown -R mysql:mysql /usr/local/mysql

sudo cp /usr/local/mysql/support-files/systemd/mariadb.service /etc/systemd/system/mariadb.service
sudo systemctl daemon-reload

Then I install DB schemas:

cd /usr/local/mysql
sudo chown -R mysql:mysql /usr/local/mysql
sudo /usr/local/mysql/scripts/mariadb-install-db --user=mysql
sudo /usr/local/mysql/bin/mariadbd-safe --user=mysql --datadir=/usr/local/mysql/data &

sudo systemctl enable mariadb
sudo /usr/local/mysql/bin/mariadb-secure-installation

I follow the instructions in "mariadb-secure-installation" and set it up. Then I add some symbolic links and edit the systemd unit file:

sudo ln -s /usr/local/mysql/bin/mariadb /usr/local/bin/mariadb
sudo ln -s /usr/local/mysql/bin/mysql /usr/local/bin/mysql
sudo nano /etc/systemd/system/mariadb.service

In my systemd unit file I enter this to show what data directory and user to use:

ExecStart=/usr/local/mysql/bin/mariadbd $MYSQLD_OPTS $_WSREP_NEW_CLUSTER $_WSREP_START_POSITION --user=mysql --datadir=/usr/local/mysql/data

Then I add some settings in the "my.cnf" file:

sudo nano /etc/my.cnf

# Adding the following settings:
[mariadb]
unix_socket=OFF
event_scheduler=ON

Then restart service and ready to go.

sudo systemctl daemon-reload
sudo kill -SIGTERM $(sudo cat $(sudo find /usr/local/mysql/data -name '*.pid' -print -quit))
sudo systemctl start mariadb
sudo systemctl enable mariadb
mariadb -u root -p

This is how I typically build and install MariaDB. First: Am I doing something wrong? (bad practices) Second: Is this enough to also start using "Primary/Secondary" db servers in a "cluster" and use MaxScale for load balancing and failover / high availability?

I can not find any code examples that tell me step-by-step how to build MariaDB from source if I want to use MaxScale later. And I find no code examples for installing MaxScale here, only in some Github repo on the video - but it's very little info there.

Any help is appreciated!

Basically... I want to have like 2-3 MariaDB servers for my Node.js App Server (web application). I want to use MaxScale for handling failovers (for high availability). In case my primary MariaDB server ("Master server") is shutdown or something, it should switch to the second server. Now I try to figure out what prerequisites I need to move on to trying my best to installing and setting up MaxScale with replication.

Thank you so much!
