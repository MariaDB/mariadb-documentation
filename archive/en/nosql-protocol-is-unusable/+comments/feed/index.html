MariaDB Knowledge Base Comments for: NoSQL Protocol is unusablehttps://mariadb.com/kb/en/nosql-protocol-is-unusable/+comments/feed/en-usMon, 05 May 2025 11:43:46 +0000Re: NoSQL Protocol is unusablehttps://mariadb.com/kb/en/nosql-protocol-is-unusable/+comments/7009<p>As Markus wrote, initial support for the 'aggregate' command has been added in 25.01.</p> <p>Yes, the lack of indexes (but for the implicit on _id) is a very known weakness. However, without support for <em>indexes on expressions</em>, it would be quite complex to implemenent. <a href="https://jira.mariadb.org/browse/MDEV-6017">MDEV-6017</a> adds basic support for indexes on expressions to MariaDB Server 11.8 and the plan is to support the indexing of fields in MaxScale 25.08.</p> <p>If there are particular commands that you find essential, please create a feature request. There is obviously a lot of territory to cover, so feedback that helps in deciding the direction to take is appreciated.</p> <p>In general, if a document database is the only thing you want, then NoSQL Protocol may not be the right choice for you. But, if you primarily use regular SQL and in addition have some needs for a document database, then it might be, as you in that case will manage with just one backend setup. With the added benefit of being able to access the document data also from the SQL side. Provided, of course, that the current limitations in the functionality are not showstoppers for you.
