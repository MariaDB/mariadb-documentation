MariaDB Knowledge Base Comments for: Best practice for open application time periodhttps://mariadb.com/kb/en/best-practice-for-open-application-time-period/+comments/feed/en-usMon, 22 Jul 2024 12:01:42 +0000Re: Best practice for open application time periodhttps://mariadb.com/kb/en/best-practice-for-open-application-time-period/+comments/5544<p>Application time periods come from <a href="https://jakewheat.github.io/sql-overview/sql-2011-foundation-grammar.html#point-in-time-2">SQL:2011</a> standard with rather constrains the implementation.</p> <p>Even documents related to the standard like <a href="https://cs.ulb.ac.be/public/_media/teaching/infoh415/tempfeaturessql2011.pdf">this article</a> tend to use similar sentinel dates.</p> <p>The <a href="/kb/en/date/">date</a> datatype supports up to 9999-12-31 which should support the current timelines of employment (for now). A date/datetime value that represents the equivalent of infinity which might have some merit as a feature request especially if there's a standard that can be referenced or an existing SQL implementation.</p> <p>Start date columns can take a <a href="/kb/en/create-table/#default-column-option">default value</a> of <a href="/kb/en/now/">now</a> (or any of its aliases).</p> Daniel Blackhttps://mariadb.com/kb/en/best-practice-for-open-application-time-period/+comments/5544
