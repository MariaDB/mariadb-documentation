MariaDB Knowledge Base Comments for: MONyog Table Alerthttps://mariadb.com/kb/en/monyog-table-alert/+comments/feed/en-usTue, 30 Jan 2024 02:19:33 +0000Re: MONyog Table Alerthttps://mariadb.com/kb/en/monyog-table-alert/+comments/2382<p>You can create a Custom SQL Object (CSO) in MONyog to monitor the table size of a particular table. You may use the below query to create a CSO,</p> <p><code class="fixed" style="white-space:pre-wrap">SELECT table_name AS `Table`, round(((data_length + index_length) / 1024 / 1024), 2) `Size_in_MB` FROM information_schema.TABLES WHERE table_schema = &#34;$DB_NAME&#34; AND table_name = &#34;$TABLE_NAME&#34;;</code></p> <p>Please specify the database_name and table_name you would wish to monitor in place of ‘$DB_NAME’ and ‘$TABLE_NAME’ in the above query.</p> <p>You may also create a CSC for the above CSO and use the below javascript, please ensure you write the CSO name in place of '&lt;name of the CSO&gt;' in the below query,</p> <pre class="fixed">function() { var obj = MONyog.UserObject('&lt;name of the CSO&gt;'); if(!obj) { return &#34;Custom SQL Object doesn't exist. Click &lt;a target='_blank' class='link' href=''&gt;here&lt;/a&gt; to add.&#34;; } if(!obj.isEnabled()) { return 'Custom SQL object for this monitor is not enabled.&lt;br&gt;Click &lt;a target=&#34;_blank&#34; class=&#34;link&#34; href=&#34;&#34;&gt;here&lt;/a&gt; to enable it.&lt;/br&gt;'; } if (!obj || !MONyog.MySQL.Custom.Available) return '(n/a)'; obj = obj.select(); var results = ''; for (i in obj) { if (results.length &gt;0) results+='&lt;br&gt;'; results += obj[i].Table + '&amp;nbsp;&amp;nbsp;&amp;nbsp;' + obj[i].Size_in_MB; } if (results.length == 0) results = 'None'; return results; } </pre><p>You may refer our documentation on creating new CSO and CSC (Custom SQL Counter for the new CSO), here &gt;&gt; <a href="http://monyogkb.webyog.com/article/67-new-csos-and-cscs">http://monyogkb.webyog.com/article/67-new-csos-and-cscs</a></p> <p>You may write to support@webyog.com if you have any queries in MONyog. There may be unused space inside the tablespace (details depends on storage engine). MySQL data folder may be mounted to another disk system than the system disk (like a RAID system/rack) and even the InnoDB tablespace may be stored at a third place.</p> <p>It is actually very difficult to provide meaningful alerts. It will also result in one more background thread running (more CPU-load, more I/O).</p> <p>So it is important to understand WHY you want this, WHAT you want to achieve and WHAT EXACTLY should be the alert condition.</p> <p>You may write to support@webyog.com if you have any queries in MONyog.
