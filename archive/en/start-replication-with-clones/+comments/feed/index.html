MariaDB Knowledge Base Comments for: Start replication with clones?https://mariadb.com/kb/en/start-replication-with-clones/+comments/feed/en-usMon, 30 Sep 2024 13:46:17 +0000Re: Start replication with clones?https://mariadb.com/kb/en/start-replication-with-clones/+comments/6470<p>Yes, shutting down and cloning the system will work, and this is a very robust method if you are able to shutdown the server briefly.</p> <p>You need to set a different server_id in the configuration file of the clone.</p> <p>You also need to configure the new slave to start replicating from the right point. Using GTID, you can do</p> <pre> CHANGE MASTER TO master_use_gtid=slave_pos, master_demote_to_slave=1, master_host=... </pre> <p>(Or master_use_gtid=current_pos if you are before <a href="/kb/en/what-is-mariadb-1010/">MariaDB 10.10</a>).</p> <p>Otherwise SHOW MASTER STATUS will show the binlog coordinates to start the slave at.</p> Kristian Nielsenhttps://mariadb.com/kb/en/start-replication-with-clones/+comments/6470
