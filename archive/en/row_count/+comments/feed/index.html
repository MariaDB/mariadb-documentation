It says</p> <p>"For INSERT ... ON DUPLICATE KEY, updated rows are counted twice. So, if INSERT adds a new rows and modifies another row, ROW_COUNT() returns 3."</p> <p>We are currently using 11.4.4. Here is the behaviour as described in the MySQL documentation:</p> <p>"For INSERT ... ON DUPLICATE KEY UPDATE statements, the affected-rows value per row is 1 if the row is inserted as a new row, 2 if an existing row is updated, and 0 if an existing row is set to its current values. If you specify the CLIENT_FOUND_ROWS flag, the affected-rows value is 1 (not 0) if an existing row is set to its current values."</p> <p>see <a href="https://dev.mysql.com/doc/refman/8.0/en/information-functions.html#function_row-count">https://dev.mysql.com/doc/refman/8.0/en/information-functions.html#function_row-count</a></p> Marc Kemmerhttps://mariadb.com/kb/en/row_count/+comments/7013
