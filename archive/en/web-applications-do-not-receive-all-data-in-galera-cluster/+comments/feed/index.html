If your application is picky about "stale" data and you want to randomly load-balance connections, you should set wsrep_causal_reads=ON. This will ensure that your application always gets what it has written - at some performance cost. Alternatively you may opt to reread the data if you can determine that it is stale, or use some sort of "sticky" balancing, routing related requests to the same node.</p> <p>Regards, Alex</p> Alex yurchenkohttps://mariadb.com/kb/en/web-applications-do-not-receive-all-data-in-galera-cluster/+comments/741Re: Web Applications do not receive all data in Galera Clusterhttps://mariadb.com/kb/en/web-applications-do-not-receive-all-data-in-galera-cluster/+comments/738<p>Hi, i think the best way to sold this is send a email about this to maria-developers@lists.launchpad.net or maria-discuss@lists.launchpad.net or open a bug request at JIRA <a href="https://mariadb.atlassian.net/">https://mariadb.atlassian.net/</a></p> <p>after problem solved return here and link the solution =)</p> roberto spadimhttps://mariadb.com/kb/en/web-applications-do-not-receive-all-data-in-galera-cluster/+comments/738
