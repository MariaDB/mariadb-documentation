MariaDB Knowledge Base Comments for: Mariadb-libs installation conflicts with existing MySQL community libshttps://mariadb.com/kb/en/mariadb-libs-installation-conflicts-with-existing-mysql-community-libs/+comments/feed/en-usTue, 20 Feb 2024 02:34:02 +0000Re: Mariadb-libs installation conflicts with existing MySQL community libshttps://mariadb.com/kb/en/mariadb-libs-installation-conflicts-with-existing-mysql-community-libs/+comments/3047<ul><li>MySQL and MariaDB both the instances can coexist. But MariaDB is to be installed to a non-standard directory. Please follow below steps to do the same:- </li></ul> <p>1) Download the compiled binary tar.gz (mariadb-5.5.24-linux-x86_64.tar.gz)</p> <p>2) Create data directory and symlinks</p> <p>a) mkdir mariadb-data</p> <p>b) ln -s mariadb-5.5.24-linux-x86_64 mariadb</p> <p>c) groupadd --system mariadb</p> <p>d) useradd -c "MariaDB Server" -d /opt/mariadb -g mariadb --system mariadb</p> <p>e) chown -R mariadb:mariadb mariadb-5.5.24-linux-x86_64/</p> <p>f) chown -R mariadb:mariadb mariadb-data/</p> <p>3) Create a new my.cnf in /opt/mariadb from support files:</p> <p>a) cp mariadb/support-files/my-medium.cnf mariadb-data/my.cnf</p> <p>b) chown mariadb:mariadb mariadb-data/my.cnf</p> <p>4) Edit the file /opt/mariadb-data/my.cnf and add custom paths, socket, port, user and the most important of all: data directory and base directory [client]</p> <p>port = 3307</p> <p>socket = /opt/mariadb-data/mariadb.sock</p> <p>[mysqld] datadir = /opt/mariadb-data</p> <p>basedir = /opt/mariadb</p> <p>port = 3307</p> <p>socket = /opt/mariadb-data/mariadb.sock</p> <p>user = mariadb</p> <p>5) Copy the init.d script from support files in the right location:</p> <p>a) cp mariadb/support-files/mysql.server /etc/init.d/mariadb</p> <p>b) chmod +x /etc/init.d/mariadb</p> <p>6) Edit /etc/init.d/mariadb </p> <p>a) replacing mysql with mariadb</p> <p>- # Provides: mysql</p> <p>+ # Provides: mariadb</p> <p>- basedir=</p> <p>+ basedir=/opt/mariadb</p> <p>- datadir=</p> <p>+ datadir=/opt/mariadb-data</p> <p>- lock_file_path="$lockdir/mysql"</p> <p>+ lock_file_path="$lockdir/mariadb"</p> <p>b) $bindir/mysqld_safe --defaults-file=/opt/mariadb-data/my.cnf --datadir="$datadir" --pid-file="$mysqld_pid_file_path" $other_args &gt;/dev/null 2&gt;&amp;1 &amp;</p> <p>c) $bindir/mysqladmin --defaults-file=/opt/mariadb-data/my.cnf ping &gt;/dev/null 2&gt;&amp;1;</p> <p>7) Run mysql_install_db</p> <p>a) cd mariadb</p> <p>b) scripts/mysql_install_db --defaults-file=/opt/mariadb-data/my.cnf</p> <p>8) Start MariaDB</p> <p>/etc/init.d/mariadb start</p> <p>9) To Make MariaDB start at system start</p> <p>a) cd /etc/init.d</p> <p>b) chkconfig --add mariadb</p> <p>c) chkconfig --levels 3 mariadb on</p> <p>10) Test MariaDB instance is running</p> <p>mysql -e "SELECT VERSION();" --socket=/opt/mariadb-data/mariadb.sock</p> <ul start="1"><li>Alternative way </li></ul> <p>Just for your information Mariadb is drop-in replacement for mysql. Therefore we can also switch from mysql to mariadb. Please perform below steps to do the same: </p> <p>1) backup your database just in case there is any issue while switching to mariadb</p> <p>2) uninstall mysql</p> <p>3) install mariadb</p> <p>4) start mariadb services All the databases and information is intact and even there is no need to have mysql dump to be performed to source into mariadb server as that data is already there</p> kanika satijahttps://mariadb.com/kb/en/mariadb-libs-installation-conflicts-with-existing-mysql-community-libs/+comments/3047
