MariaDB Knowledge Base Comments for: maxscale 6 schemarouter about start transactionhttps://mariadb.com/kb/en/maxscale-6-schemarouter-about-start-transaction/+comments/feed/en-usTue, 19 Mar 2024 05:42:08 +0000Re: maxscale 6 schemarouter about start transactionhttps://mariadb.com/kb/en/maxscale-6-schemarouter-about-start-transaction/+comments/5508<p>The problem with <code>START TRANSACTION</code> is that the schemarouter cannot know which node to start the transaction on. You could use <code>SET autocommit=0</code> which is routed to all nodes in the cluster as it modifies the session state. This should allow you to start a transaction without knowing which node it will end up on. You can commit the transaction by enabling autocommi with <code>SET autocommit=1</code> which should also be routed to all nodes.</p> Markus Mäkelähttps://mariadb.com/kb/en/maxscale-6-schemarouter-about-start-transaction/+comments/5508
