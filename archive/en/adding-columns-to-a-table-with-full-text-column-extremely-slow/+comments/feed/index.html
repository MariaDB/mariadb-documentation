To add insult to the injury, if an <code class="fixed" style="white-space:pre-wrap">ALTER TABLE</code> operation would involve creating multiple <code class="fixed" style="white-space:pre-wrap">FULLTEXT INDEX</code> (like it would if the table were to be rebuilt for adding the columns), then InnoDB would refuse a native operation, and the inefficient <code class="fixed" style="white-space:pre-wrap">ALGORITHM=COPY</code> would be used.</p> <p>You could try dropping all fulltext indexes first, and then perform <code class="fixed" style="white-space:pre-wrap">ALTER TABLE t FORCE, ADD COLUMN …, ADD FULLTEXT INDEX …, ALGORITHM=INPLACE</code> (creating at most one fulltext index). The <code class="fixed" style="white-space:pre-wrap">FORCE</code> keyword prevents <code class="fixed" style="white-space:pre-wrap">ALGORITHM=INSTANT</code> operation and forces a rebuild. It should not be necessary, but I did not test it.</p> <p>If there are multiple fulltext indexes, you can create them one at a time with subsequent <code class="fixed" style="white-space:pre-wrap">ALTER TABLE t ADD FULLTEXT INDEX</code> statements. Note: when you add the first fulltext index, a hidden <code class="fixed" style="white-space:pre-wrap">FTS_DOC_ID</code> column will be created inside InnoDB. So, typically adding the first fulltext index will require the table to be rebuilt. Also, note that when any <code class="fixed" style="white-space:pre-wrap">FULLTEXT</code> or <code class="fixed" style="white-space:pre-wrap">SPATIAL</code> index are to exist after the <code class="fixed" style="white-space:pre-wrap">ALTER TABLE</code>, the operation cannot be executed online (while allowing concurrent writes to the table).</p> Marko Mäkelähttps://mariadb.com/kb/en/adding-columns-to-a-table-with-full-text-column-extremely-slow/+comments/4773Re: Adding columns to a table with full text column extremely slowhttps://mariadb.com/kb/en/adding-columns-to-a-table-with-full-text-column-extremely-slow/+comments/4771<p>Hello -</p> <p>This is using InnoDB.</p> <p>Here is part of the output from my script,without any full-text columns - the numbers in brackets like [0.007926] is the timing in seconds.</p> <pre class="fixed"> &gt;&gt;&gt; Create / modify columns &gt;&gt;&gt; Module display name &gt;&gt;&gt; Adding the column ALTER TABLE `wdeModule` ADD COLUMN `displayName` varchar(191) NOT NULL DEFAULT ? &lt;&lt;&lt; Adding the column [0.007926] &gt;&gt;&gt; Creating b-tree index CREATE INDEX `idx_displayName_btree` ON `wdeModule` (`displayName`) &lt;&lt;&lt; Creating b-tree index [0.034811] &lt;&lt;&lt; Module display name [0.043111] &gt;&gt;&gt; List of records page &gt;&gt;&gt; Adding the column ALTER TABLE `wdeModule` ADD COLUMN `listOfRecordsPageId` int(10) unsigned NOT NULL DEFAULT ?
