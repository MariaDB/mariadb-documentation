I noted more recently a very negative impact of having too many persistent connections and when using the setting:</p> <p>thread_definition_cache = -1;</p> <p>That combined with the various table caches was taking up a bunch of space. It seems the thread_definition_cache, over the period of a few days went to the max value, and never aged itself out. So, the system memory use exploded till OOM.</p> <p>I ended up flushing the hosts and reducing the various caching variables until the system was healthy again. We also always mark in the "KB" in what release a specific feature was added. This means that when one wants to read up on a feature, one can quickly skip the parts that does not affect the version that is used. In addition one can quickly see the benefits one can get by upgrading MariaDB.</p> <p>We don't have the resources to maintain a manual per release.
