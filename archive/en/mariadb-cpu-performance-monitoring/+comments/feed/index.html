MariaDB Knowledge Base Comments for: MariaDB CPU Performance Monitoringhttps://mariadb.com/kb/en/mariadb-cpu-performance-monitoring/+comments/feed/en-usMon, 07 Apr 2025 14:28:47 +0000Re: MariaDB CPU Performance Monitoringhttps://mariadb.com/kb/en/mariadb-cpu-performance-monitoring/+comments/4769<p>You certainly can!</p> <p>As to how to go about it, I found that what works best for me is <a href="https://www.navicat.com/en/products/navicat-monitor">Navicat Monitor</a>. It's an agent-less database server instance monitoring tool for MySQL and MariaDB. Version 2 adds support for SQL Server as well. It can monitor database process and system resources for locally hosted database instances as well as those provided via cloud services such as Amazon Web Services (AWS), Oracle Cloud, Google Cloud and Microsoft Azure.</p> <p>On Windows Type servers, you can configure CPU &amp; Memories section of the Instance Configuration Dialog to monitor O/S metrics over Simple Network Management Protocol (SNMP). That will cause server metrics like CPU, Memory, and Disk Usage to appear in Dashboard Instance Cards. From there, you can click on the system metrics to see more details, including Swap Usage, Connections, and Network Throughput. Each metric includes an interactive chart.</p> <p>Give that a try!</p> <p>Rob G</p> Rob Gravellehttps://mariadb.com/kb/en/mariadb-cpu-performance-monitoring/+comments/4769Re: MariaDB CPU Performance Monitoringhttps://mariadb.com/kb/en/mariadb-cpu-performance-monitoring/+comments/4760<p>Before you potentially prematurely update your hardware, enable the slow query log <a href="https://mariadb.com/kb/en/slow-query-log/">https://mariadb.com/kb/en/slow-query-log/</a>. Also include SHOW GLOBAL STATUS; SHOW GLOBAL VARIABLES; SHOW ENGINE INNODB STATUS; from at least 24 hrs of uptime. Include current my.cnf settings as well.</p> <p>CPU stats themselves don't often lead to something that can be directly enacted upon.</p> Daniel Blackhttps://mariadb.com/kb/en/mariadb-cpu-performance-monitoring/+comments/4760
