MariaDB Knowledge Base Comments for: Mariadb Replication Questionhttps://mariadb.com/kb/en/mariadb-replication-question/+comments/feed/en-usMon, 25 Nov 2024 11:10:45 +0000Re: Mariadb Replication Questionhttps://mariadb.com/kb/en/mariadb-replication-question/+comments/6632<p>Yes, by default a DROP DATABASE xxx on the master will be replicated to the slave and delete the database there as well.</p> <p>One way to avoid this is to disable binlogging for the DROP DATABASE, so that it does not participate in the replication, using sql_log_bin: <code class="unknown_macro">&lt;&lt;<span class="macro_name">code</span><span class="macro_arg_string"> lang=mysql</span>&gt;&gt;</code> SET STATEMENT sql_log_bin=0 FOR DROP DATABASE xxx; <code class="unknown_macro">&lt;&lt;<span class="macro_name">code</span><span class="macro_arg_string"></span>&gt;&gt;</code> It is best to test this first by creating an extra empty database and test that it can be dropped on the master without affecting the slave.</p> Kristian Nielsenhttps://mariadb.com/kb/en/mariadb-replication-question/+comments/6632
