MariaDB Knowledge Base Comments for: Move Maria DB from one server to other other server with LVM snapshothttps://mariadb.com/kb/en/move-maria-db-from-one-server-to-other-other-server-with-lvm-snapshot/+comments/feed/en-usMon, 10 Feb 2025 16:27:45 +0000Re: Move Maria DB from one server to other other server with LVM snapshothttps://mariadb.com/kb/en/move-maria-db-from-one-server-to-other-other-server-with-lvm-snapshot/+comments/5155<p>LVM is a suitable tool because it uses consistent snapshot and as such won't corrupt the data directory (like rsync/cp will).</p> <p>Having LVM snapshot there can create a little more IO work for the OS/MariaDB so recommend testing a few time and observing the impact to see if its within acceptable parameters.</p> <p>If your second server Y intends to server as a replica, you may need to FLUSH TABLES WITH READ LOCK, observe binlog positions, LVM snapshot, release FLUSH TABLES lock. The binlog position will be needed to create the replica.</p> <p>When the server Y starts up, there will be an element of crash recovery that will rollback transactions and apply completed transactions to the table space. This is ok and expected.</p> <p>SET GLOBAL innodb_max_purge_lag_wait=0, if supported on your MariaDB version (see <a href="https://jira.mariadb.org/browse/MDEV-16952">MDEV-16952</a>), a little time before the snapshot will reduce the apply completed transactions step and can be set back to default after the snapshot.</p> Daniel Blackhttps://mariadb.com/kb/en/move-maria-db-from-one-server-to-other-other-server-with-lvm-snapshot/+comments/5155
