Filterhttps://mariadb.com/kb/en/mariadb-maxscale-6-named-server-filter/+comments/5816<p>If you added <code>filters=NamedServerFilter</code> under the <code>[Read-Write-Service]</code> section, that should already cause the filter to be linked to the service.</p> <p>The output of <code>maxctrl show filters</code> should have the number of queries that were diverted by the filter. If the number increases, you know the filter works. An alternative way is to enable the info level log by adding <code>log_info=true</code> under the <code>[maxscale]</code> section and then looking at the log to see where the queries are routed.</p> Markus Mäkelähttps://mariadb.com/kb/en/mariadb-maxscale-6-named-server-filter/+comments/5816Re: Named Server Filterhttps://mariadb.com/kb/en/mariadb-maxscale-6-named-server-filter/+comments/5815<p>After adding the [NamedServerFilter] and including it in the [Read-Write-Service] in the maxscale.cnf file, and restarting the service, I could see the Filter show in the web UI, but it was not connected to the service...</p> <p>I had to manually link the filter to the service in the web GUI. </p> <p>Also, is there a way to test the filter is working?
