Re: Using HeidiSQL with MariaDB AX Columnstore 10.2.10 Columnstore 1.1.2-1

For MariaDB ColumnStore we would typically recommend using the cpimport tool to insert CSV files. But if you are using a tool that does LOAD DATA INFILE please make sure it is not as part of a transaction and autocommit is on. Otherwise it gets inserted as single inserts instead of a bulk insert and will be very slow.

If you do this inserts of that size should only take a few seconds and larger inserts will likely be much faster than inserting into InnoDB.
