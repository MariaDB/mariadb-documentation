MariaDB Knowledge Base Comments for: How to access MariaDB from a Linux systemd service without passwordhttps://mariadb.com/kb/en/how-to-access-mariadb-from-a-linux-systemd-service-without-password/+comments/feed/en-usMon, 03 Mar 2025 12:22:09 +0000Re: How to access MariaDB from a Linux systemd service without passwordhttps://mariadb.com/kb/en/how-to-access-mariadb-from-a-linux-systemd-service-without-password/+comments/6717<p>The systemd service has <a href="https://www.freedesktop.org/software/systemd/man/latest/systemd.exec.html#ProtectHome=">ProtectHome=yes</a> by default making the /root/.my.cnf file inaccessible to the systemd service.</p> <p>While disabling with ProtectHome=false may seem like the easy option, consider the alternative:</p> <pre class="fixed">ALTER USER root@localhost IDENTIFIED VIA unix_socket </pre><p>Ref: <a href="/kb/en/authentication-plugin-unix-socket/#security">unix socket authentication</a>.</p> <p>With this the /root/.my.cnf file is no longer needed, and the root unix user is the only one that can access the root@localhost MariaDB user account.</p> <p>If your systemd service runs under a different user, say xyzservice, then;</p> <pre class="fixed">CREATE USER xyservice@localhost IDENTIFIED VIA unix_socket </pre><p>And then provide the sufficient <a href="/kb/en/grant/">grants</a> to that user based on what the service needs.</p> Daniel Blackhttps://mariadb.com/kb/en/how-to-access-mariadb-from-a-linux-systemd-service-without-password/+comments/6717
