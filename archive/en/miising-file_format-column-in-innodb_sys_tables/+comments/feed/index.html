MariaDB Knowledge Base Comments for: Miising FILE_FORMAT column in INNODB_SYS_TABLEShttps://mariadb.com/kb/en/miising-file_format-column-in-innodb_sys_tables/+comments/feed/en-usMon, 05 Aug 2024 11:58:34 +0000Re: Missing FILE_FORMAT column in INNODB_SYS_TABLEShttps://mariadb.com/kb/en/miising-file_format-column-in-innodb_sys_tables/+comments/4636<p><code>INFORMATION_SCHEMA.INNODB_SYS_TABLES</code> is a hard-coded view that is provided to the InnoDB internal data dictionary table <code>SYS_TABLES</code> that resides in the system tablespace. The column <code>FILE_FORMAT</code> does not have any direct counterpart in <code>SYS_TABLES</code>.</p> <p>The configuration parameters <code>innodb_file_format</code> and <code>innodb_large_prefix</code> were deprecated in <a href="/kb/en/mariadb-1022-release-notes/">MariaDB 10.2.2</a> when the InnoDB changes from MySQL 5.7.9 were merged. The motivation for the deprecation and removal was that originally these parameters were introduced to supposedly allow downgrading to older versions than MySQL 5.5. To make matters worse, the default values of these parameters caused the files to remain in the backward-compatible format. Only MySQL 5.7 and <a href="/kb/en/what-is-mariadb-102/">MariaDB 10.2</a> would change the defaults to sane values and deprecate these useless parameters. (The predecessors of MySQL 5.5 were already EOL at that point, and besides, we do not really support downgrades.)</p> <p>Those 2 deprecated parameters were removed in <a href="/kb/en/what-is-mariadb-103/">MariaDB 10.3</a>, but they were put back later in <a href="https://jira.mariadb.org/browse/MDEV-18399">MDEV-18399</a>, because the sane defaults were explicitly specified in commonly used configuration files.</p> <p>I do not think that it makes sense to carry the column in <code>INNODB_SYS_TABLES</code>.</p> <p>Note: In <a href="/kb/en/what-is-mariadb-103/">MariaDB 10.3</a> and <a href="/kb/en/what-is-mariadb-104/">MariaDB 10.4</a>, InnoDB file formats were extended by <a href="https://jira.mariadb.org/browse/MDEV-11369">MDEV-11369</a> and <a href="https://jira.mariadb.org/browse/MDEV-15562">MDEV-15562</a>. You can disable those format changes by</p> <pre class="fixed">SET GLOBAL innodb_instant_alter_column_allowed=never; </pre><p>which was implemented in <a href="https://jira.mariadb.org/browse/MDEV-20590">MDEV-20590</a>.</p> Marko Mäkelähttps://mariadb.com/kb/en/miising-file_format-column-in-innodb_sys_tables/+comments/4636
