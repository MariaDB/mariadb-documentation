MariaDB Knowledge Base Comments for: Migrating from PostgreSQL to MariaDBhttps://mariadb.com/kb/en/migrating-from-postgresql-to-mariadb/+comments/feed/en-usMon, 02 Dec 2024 12:26:52 +0000Re: Migrating from POstgre SQL to Maria DBhttps://mariadb.com/kb/en/migrating-from-postgresql-to-mariadb/+comments/3741<p>You may have to manually convert the schema to work with MariaDB, but one way to migrate the data is to create foreign tables in MariaDB that reference the tables in PostgreSQL using <a href="/kb/en/connect/">CONNECT storage engine</a> along with a PostgreSQL <a href="/kb/en/connect-odbc-table-type-accessing-tables-from-another-dbms/">ODBC</a> or <a href="/kb/en/connect-jdbc-table-type-accessing-tables-from-another-dbms/">JDBC</a> driver. And then you could do something like:</p> <pre class="fixed">INSERT INTO local_tab SELECT * FROM foreign_tab; </pre><p>You could also do the opposite and use <a href="https://github.com/EnterpriseDB/mysql_fdw">mysql_fdw</a> to create foreign tables on the PostgreSQL side that reference tables in MariaDB. And then you could do something like:</p> <pre class="fixed">INSERT INTO foreign_tab SELECT * FROM local_tab; </pre>Geoff Monteehttps://mariadb.com/kb/en/migrating-from-postgresql-to-mariadb/+comments/3741
