If you remove the .my-healthcheck.sh and have MARIADB_AUTO_UPGRADE=1 then the .my-healthcheck.cnf will be regenerated with the currently configured port.</p> <p>The healtcheck user/password will be regenerated in the server also with this option.</p> Daniel Blackhttps://mariadb.com/kb/en/using-healthcheck-sh/+comments/6761Using Healthcheck.sh With Custom Port (Docker)https://mariadb.com/kb/en/using-healthcheck-sh/+comments/6749<p>I had issues using <code>healthcheck.sh</code> as suggested by Bren X., because I set up MariaDB with a custom port 3307 directly within the MariaDB configuration, such as:</p> <pre> [mariadb] port = 3307 </pre> <p>This custom port collides with the default <code>healthcheck.sh</code> in Docker, that uses a file <code>/var/lib/mysql/.my-healthcheck.cnf</code> generated similar to this:</p> <pre> [mariadb-client] port=3306 socket=/run/mysqld/mysqld.sock user=healthcheck password=... protocol=tcp </pre> <p>This file is passed as <code>--defaults-extra-file</code> to the MariaDB client.</p> <p>Overriding this file is not an option, as it contains a generated password.</p> <p>What works, but is undocumented here, is, that <code>healthcheck.sh</code> also accepts parameters <code>--defaults-file</code> and <code>--defaults-extra-file</code>. By experiment, as the MariaDB client documentation is not that clear to me, it works passing <code>--defaults-file</code> with reference to a file like this:</p> <pre> [mariadb-client] port=3307 </pre> <p>It will override the port from the extra-file.</p> <p>Now, extending the <code>test</code> section as given by Bren X., this extended <code>healthcheck.sh</code> call would work for a file mounted to, for example, <code>/etc/mysql/conf.d/custom-client.cnf</code>:</p> <pre> test: [ "CMD", "healthcheck.sh", "--su-mysql", "--defaults-file=/etc/mysql/conf.d/custom-client.cnf" "--connect", "--innodb_initialized" ] </pre> <p>There may be better options, like just sticking to the default port 3306 internally and use Docker to map it to 3307 to the outside. But at least, this approach works, and of course could also be used to pass other parameters to the MariaDB client.</p> Mark Michaelishttps://mariadb.com/kb/en/using-healthcheck-sh/+comments/6749Re: Using Healthcheck.shhttps://mariadb.com/kb/en/using-healthcheck-sh/+comments/6578<p>To use this script in docker-compose.yml:</p> <pre> services: db: image: mariadb:10.11 restart: always healthcheck: interval: 30s retries: 3 test: [ "CMD", "healthcheck.sh", "--su-mysql", "--connect", "--innodb_initialized" ] timeout: 30s volumes: - mariadb:/var/lib/mysql </pre> <p><a href="https://docs.docker.com/engine/reference/builder/#healthcheck">https://docs.docker.com/engine/reference/builder/#healthcheck</a></p> Bren X.https://mariadb.com/kb/en/using-healthcheck-sh/+comments/6578
