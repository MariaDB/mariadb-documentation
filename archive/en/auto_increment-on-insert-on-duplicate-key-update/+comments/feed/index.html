ON DUPLICATE KEY UPDATEhttps://mariadb.com/kb/en/auto_increment-on-insert-on-duplicate-key-update/+comments/5386<p>The <a href="/kb/en/last_insert_id/">last_insert_id</a> returns only the last inserted value which is intended. Hitting an "ON DUPLICATE KEY UPDATE" doesn't count as an insertion so returns the value used in the last successful insert.</p> <p><a href="/kb/en/auto_increment/#missing-values">auto_increment</a> in cases like this can contain missing values. Its part of the architecture that its easier to allocate an auto_increment value before it detected that a duplicate occurs.</p> Daniel Blackhttps://mariadb.com/kb/en/auto_increment-on-insert-on-duplicate-key-update/+comments/5386
