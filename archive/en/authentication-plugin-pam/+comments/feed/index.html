Holzgraefehttps://mariadb.com/kb/en/authentication-plugin-pam/+comments/1361Re: PAM authentication pluginhttps://mariadb.com/kb/en/authentication-plugin-pam/+comments/1360<p>Well, the "can only check the users own password unless invoked by root" limitation is there for a reason:</p> <p>as unix_chkpwd is a local binary and not a service it can be used to perform brute force dictionary attacks against /etc/shadow without a need to have read access to the shadow file otherwise ...</p> <p>Restricting access to this for root isn't necessary as root can read the shadow file (and other things like changing user passwords) anyway ...</p> <p>But for any other user starting unix_chkpwd extra care is taken that only the password of this single user can be cracked this way (again not that much harm with this as an attacker must somehow gained access to that user account already anyway)</p> <p>Any shadow entry for other users are still hidden by the inner working of unix_chkpwd </p> pluginhttps://mariadb.com/kb/en/authentication-plugin-pam/+comments/1357<p>Doesn't work for me (Ubuntu 14.10, Ubuntu distribution MariaDB packages)</p> <p>"Problem" is that unix_chkpwd explicitly checks that its UID matches the username given as cmdline argument, *unless* the UID is 0 (root). As it checks UID, not EUID, suid chmod magic doesn't make a difference here.</p> <p>The related check code in unix_chkpwd.c is at <a href="https://git.fedorahosted.org/cgit/linux-pam.git/tree/modules/pam_unix/unix_chkpwd.c#n129">https://git.fedorahosted.org/cgit/linux-pam.git/tree/modules/pam_unix/unix_chkpwd.c#n129</a></p> <p> if (getuid() == 0) { user=argv[1]; } else { user = getuidname(getuid()); /* if the caller specifies the username, verify that user matches it */ if (strcmp(user, argv[1])) { user = argv[1]; /* no match -&gt; permanently change to the real user and proceed */ if (setuid(getuid()) != 0) return PAM_AUTH_ERR; } }</p> <p>So unless mysqld is running as root the "does the given user name match my UID?" check will only succeed if the mysql user name matches the system user name, e.g. usually pam_unix will work for mysql users "mysql@..." if mysqld is running as unix user "mysql"</p> Hartmut Holzgraefehttps://mariadb.com/kb/en/authentication-plugin-pam/+comments/1357Re: PAM authentication pluginhttps://mariadb.com/kb/en/authentication-plugin-pam/+comments/1121<p>mysql_native_password plugin.</p> Federico Razzolihttps://mariadb.com/kb/en/authentication-plugin-pam/+comments/1121Re: PAM authentication pluginhttps://mariadb.com/kb/en/authentication-plugin-pam/+comments/1120<p>Details on how to revert back to normal authentication would be very handy</p> Stephanie Lockehttps://mariadb.com/kb/en/authentication-plugin-pam/+comments/1120Re: PAM authentication pluginhttps://mariadb.com/kb/en/authentication-plugin-pam/+comments/876<p>Could it be a MariaDB version issue, as the system I'm testing this on is only at 5.2.14?
