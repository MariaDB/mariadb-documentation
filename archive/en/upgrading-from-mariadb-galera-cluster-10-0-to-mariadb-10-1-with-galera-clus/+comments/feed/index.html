MariaDB Knowledge Base Comments for: Upgrading from MariaDB Galera Cluster 10.0 to MariaDB 10.1 with Galera Clusterhttps://mariadb.com/kb/en/upgrading-from-mariadb-galera-cluster-10-0-to-mariadb-10-1-with-galera-clus/+comments/feed/en-usMon, 23 Dec 2024 04:10:31 +0000Re: Upgrading from MariaDB Galera Cluster 10.0 to MariaDB 10.1 with Galera Clusterhttps://mariadb.com/kb/en/upgrading-from-mariadb-galera-cluster-10-0-to-mariadb-10-1-with-galera-clus/+comments/4176<p>Following the upgrade procedure step by step I cam across an issue on Ubuntu (and therefore also Debian): The installation of the new packages will automatically start MariaDB. At that point the configs in /etc/mysql/* were not adjusted. To better handle this, it should be noted that automatic daemon starts should be rpevented. A quick and easy way to do is is by using the policy-rc.d:</p> <hr> <pre>tee /usr/sbin/policy-rc.d &lt;&lt;EOF !/bin/sh exit 101 EOF chmod +x /usr/sbin/policy-rc.d </pre> <hr> <p>Once the installation of the packages is done, the configs can be adjusted and then MariaDB can be started as outlined in the guide. Afterwards /usr/sbin/policy-rc.d can be removed again.</p> <p>It should also be noted that mysql_upgrade is not really necessary on default Ubuntu/Debian systems as the included /etc/mysql/debian-start script already does a mysql_upgrade during startup.</p> Claudio Kuenzlerhttps://mariadb.com/kb/en/upgrading-from-mariadb-galera-cluster-10-0-to-mariadb-10-1-with-galera-clus/+comments/4176Re: Upgrading from MariaDB Galera Cluster 10.0 to MariaDB 10.1https://mariadb.com/kb/en/upgrading-from-mariadb-galera-cluster-10-0-to-mariadb-10-1-with-galera-clus/+comments/3101<p>On RHEL7.4 at least systemd-219 does not understand TimeoutSec=infinity, TimeoutSec=0 works much better.</p> Martin Mortensenhttps://mariadb.com/kb/en/upgrading-from-mariadb-galera-cluster-10-0-to-mariadb-10-1-with-galera-clus/+comments/3101
